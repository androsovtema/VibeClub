# 14 — Соответствие законам РФ (аудит стека, 2026-07-13)

Контекст решения: платформа в первую очередь под российский рынок. Стек —
Supabase Cloud (EU), GitHub Pages, Resend, Umami Cloud — весь иностранный.
Вопрос Тёмы: что соблюдаем, что нарушаем, куда сворачивать.

**TL;DR-вердикт: стек оставляем на запуск, но с тремя оговорками.**
1. До анонса — подать два бесплатных уведомления в РКН и закрыть T20
   (политика + согласия): это снимает самые дешёвые и самые вероятные штрафы.
2. Главное структурное нарушение — **локализация** (ПДн россиян собираются
   сразу в EU-базу Supabase). Лечится только переносом БД в РФ. План — этап 2,
   триггер — живой рост, а не дата.
3. OAuth «Войти через Google/Apple» **не добавлять никогда** (и так решено в T1) —
   с июля 2026 за это штрафуют.

## Карта законов, которые нас касаются

| Закон / норма | Суть | Нас касается? |
|---|---|---|
| 152-ФЗ ст. 18 ч. 5 — локализация | Первичный сбор ПДн граждан РФ — только в БД на территории РФ; с 01.07.2025 прямой запрет первичного сбора в иностранные хранилища | **Да — нарушаем** (Supabase EU) |
| 152-ФЗ ст. 22 — уведомление РКН об обработке | Подаётся до начала обработки, касается и физлиц/ИП/самозанятых | **Да — не подано** |
| 152-ФЗ ст. 12 — трансграничная передача | Отдельное уведомление РКН до начала передачи за рубеж | **Да — не подано** (Supabase DE, Resend US, Umami) |
| 152-ФЗ — согласие, политика | Политика конфиденциальности на сайте, согласия на обработку | **Да — закрывает T20** (усилить: чекбоксы согласия) |
| Штрафы за утечки (с 30.05.2025) | Утечка 1–10 тыс. субъектов: 3–5 млн ₽; неуведомление об утечке за 24 ч: 1–3 млн ₽; повторная крупная — оборотный до 500 млн ₽ | Да — мотивация не копить лишние ПДн |
| 149-ФЗ + КоАП 13.55 (штрафы с июля 2026) | Запрет авторизации на росс. сайтах через иностранные OAuth-сервисы (Google/Apple ID). Email+пароль с любой почтой (включая gmail) — разрешён. Для сайтов физлиц есть исключение | **Нет нарушения** — OAuth не используем. Не добавлять |
| 149-ФЗ ст. 10.2-1 — рекомендательные технологии | Правила для персонализированных рекомендаций | Не применяется — витрина без персонализации (сортировка одна для всех). Следить при добавлении «рекомендуем тебе» |
| 149-ФЗ ст. 10.1 — реестр ОРИ | Сервисы обмена сообщениями обязаны хранить сообщения в РФ | Комментарии формально близко; на практике РКН вносит по своему требованию. Риск при масштабе, не сейчас |
| Маркировка рекламы (ЕРИР) | Любая реклама на сайте маркируется | Пока рекламы нет; вспомнить при монетизации |

## Что нарушаем сейчас (по убыванию вероятности прилёта)

1. **Нет уведомления РКН об обработке ПДн.** Самое частое и легко проверяемое
   нарушение (РКН сверяет сайты с реестром операторов). Штраф: граждане
   5–10 тыс. ₽, ИП/должностные 30–50 тыс. ₽, юрлица 100–300 тыс. ₽.
   Подача бесплатна: `pd.rkn.gov.ru` → электронная форма. Оператор сейчас —
   Тёма-физлицо (публичный сервис ≠ «личные/семейные нужды», исключение не
   работает).
2. **Нет уведомления о трансграничной передаче.** Данные (email, имя, аватар)
   уходят в Германию (Supabase, AWS eu-central-1), США (Resend, Umami Cloud).
   Штраф ИП/юрлицам 100–300 тыс. ₽. Тоже бесплатная форма, подаётся один раз
   со списком стран.
3. **Нет политики конфиденциальности и явных согласий** на сайте. T20 закрывает
   страницу; дополнительно нужны чекбоксы согласия на форме регистрации и
   упоминание счётчика Umami.
4. **Локализация: первичный сбор ПДн — сразу в EU.** Это нарушение с 2015 года
   (за него блокировали LinkedIn), с 01.07.2025 — прямой запрет. Согласие
   пользователя НЕ снимает требование: сначала запись в базу на территории РФ,
   потом хоть куда с уведомлением. Штраф юрлицам 1–6 млн ₽ (повторно до 18 млн);
   к физлицам практики почти нет, но при заметности проекта (публичный анонс,
   рост) риск растёт. Уведомлениями не лечится — только переносом БД.

## Что в порядке

- **Email+пароль и магик-линк** — легальны, почта пользователя любая (gmail ок).
  Запрет касается OAuth-кнопок иностранных сервисов — их нет и не будет (T1).
- **Umami без cookies** — cookie-баннер не нужен; данные о визитах обезличены
  (IP не сохраняется в сыром виде). Трансграничку всё равно указать в уведомлении.
- **GitHub Pages** — статика, ПДн через неё не собираются (формы пишут напрямую
  в Supabase). Нарушения нет; риск только инфраструктурный — доступность GitHub
  из РФ, держим в голове как и для любого иностранного сервиса.
- **Минимизация данных уже по дизайну**: телефон/почта профиля — опциональные
  публичные поля по желанию пользователя, паспортов/платежей нет. Меньше ПДн —
  меньше цена утечки.

## План по этапам

**Этап 0 — до анонса (руками Тёмы, ~час):**
- [ ] Уведомление РКН об обработке ПДн (`pd.rkn.gov.ru`, электронная форма).
- [ ] Уведомление о трансграничной передаче (страны: Германия, США).
- [ ] T20 + юр-усиление: политика конфиденциальности (что собираем, где храним —
      честно указать зарубежные сервисы, как удалить), чекбокс согласия при
      регистрации, упоминание Umami.

**Этап 1 — первые живые сотни пользователей:**
- [ ] Перенести БД в РФ: self-hosted Supabase (docker) на российском VPS
      (Timeweb/Selectel/Yandex Cloud, ~500–1500 ₽/мес). Код не меняется —
      только `SUPABASE_URL` и ключи в `js/config.js`, данные — дамп/restore.
      Это закрывает локализацию целиком.
- [ ] SMTP → российский сервис (Unisender Go / Mailganer / DashaMail) вместо
      Resend, домен `mail.wedesignerz.com` переезжает.
- [ ] Umami → self-host на том же VPS (скрипт и id меняются в одном месте —
      `js/analytics.js`/`js/config.js`) или Яндекс.Метрика (тогда cookie-политика).
- Статику можно оставить на GitHub Pages — она ПДн не трогает.

**Этап 2 — масштаб (тысячи пользователей / монетизация):**
- Оформление ИП/юрлица (оборотные штрафы и договоры того потребуют раньше законов).
- Пересмотр ОРИ-риска для комментариев/чата, маркировка рекламы при монетизации.
- Ревизия этого дока: законы меняются несколько раз в год, дата аудита в шапке.

## Почему не переезжаем прямо сейчас

Сейчас в базе 3 пользователя и 3 проекта — все свои. Цена нарушения локализации
на этом масштабе ≈ 0, а переезд до анонса съест недели, которые нужны запуску
(e2e, T20, сидинг). Уведомления РКН (этап 0) при этом подаём честно и сразу:
они дешёвые, быстрые и закрывают самые реальные штрафы. Формально указать
в уведомлении текущее место хранения (Германия) — законно как минимум в части
трансграничной передачи; остаточный риск по локализации осознан и ограничен
этапом 1.
