# 08 — Как мы работаем (принцип и роли)

Этот файл — точка входа для любой новой сессии. Если ты (или новая модель) открыл проект
и не знаешь контекста — читай отсюда, потом `docs/README.md`.

## Роли

**Тёма — владелец продукта и вайбкодер.**
Держит видение, принимает решения, придумывает и направляет. Пишет на естественном языке
по-русски, не обязан знать код. Смотрит результат в браузере и говорит, что не так.
Делает то, что требует его личности/кредов (аккаунты, ключи, деплой, Telegram, видео).

**Opus 4.8 (я) — «мозг» проекта.**
Не пишу продуктовый код (кроме мелких точечных правок). Моя работа:
- стратегия и позиционирование, архитектура, схема БД и безопасность (RLS);
- вся документация в `docs/`;
- разбиваю работу на задачи для Sonnet с критериями приёмки;
- ревью результата, диагностика, план дальнейших шагов;
- задаю уточняющие вопросы, когда решение за Тёмой; иначе беру разумный дефолт и иду.
Общаюсь с Тёмой по-русски, коротко и по делу, без воды.

**Sonnet — исполнитель продуктовых задач.**
Пишет код строго по задачам из `docs/04-tasks-sonnet.md`. Одна задача за раз,
до критериев приёмки. Не меняет архитектуру и позиционирование — они заданы
в `docs/`.

**Terra — исполнитель инфраструктурных и cutover-задач.**
Получает отдельные самодостаточные промпты из `docs/prompts/terra/`. Terra
сначала делает безопасную repo-часть, а команды с VPS/секретами выполняет Тёма
строго по `infra/RUNBOOK.md`. Одна задача за раз; никаких скрытых deploy/push.

## Как течёт работа (цикл)

```
Тёма (идея / правка на русском)
   → Opus: обсуждает, решает, обновляет docs/, пишет задачу + детальный промпт
   → Тёма: копирует промпт в сессию нужного исполнителя (Sonnet или Terra)
   → Sonnet/Terra: выполняет одну документированную задачу
   → Тёма/Opus: смотрят в браузере (localhost:8080), ревью
   → правки → повтор, пока не примем
```

Opus и исполнители **не общаются напрямую**. Интерфейс между ними —
документация в `docs/` и готовый промпт, который Тёма переносит. Поэтому
задачи пишутся самодостаточно: с путями, контекстом и критериями приёмки.

## Принципы

- **Русский язык** во всём общении (см. `CLAUDE.md`).
- **Честность данных:** никаких выдуманных цифр/участников на проде. Только реальное из БД
  или честная формулировка. (См. `00-vision.md`.)
- **Не шаблонно:** сайт не должен выглядеть как типовой AI-лендинг (см. `06`, раздел
  «Анти-шаблон»).
- **Безопасность:** anon-ключ публичен, вся защита на RLS. `service_role` никогда на фронте.

## Как пользоваться этим в новой сессии

1. Прочитать `CLAUDE.md` (авто) → `docs/08-workflow.md` (этот) → `docs/README.md`.
2. Посмотреть текущий статус и «Порядок и зависимости» в `docs/04-tasks-sonnet.md`.
3. Продолжать с ближайшей незакрытой задачи.
