# 13 — Дизайн-система: ритм, контейнеры, консистентность

Дата: 2026-07-08. Итог аудита всей верстки + правила, по которым теперь живут страницы.
Токены — в `css/tokens.css`, это единственный источник правды.

## Что было не так (аудит)

1. **Глобальное `section { padding: 180px 80px }`** — каждая страница боролась с ним
   частичными переопределениями (`padding-top` меняли, `padding-bottom` забывали).
   Результат: на `/me` между формой и «Твоими проектами» было ~300px пустоты,
   на `/profile` — те же 300px между профилем и проектами.
2. **Три горизонтальные сетки одновременно**: шапка/hero/футер — 28px, секции — 80px,
   разные блоки одной страницы — в контейнерах 860px и 1200px вперемешку.
   «Назад», имя и заголовки стояли на разных X.
3. **Микро-разнобой**: 30+ уникальных font-size (включая 14.5/13.5/12.5px),
   радиусы 4/5/6/7/10px мимо токенов, два размера инпутов в одной форме.
4. **Логика страниц**: имя в шапке вело в кабинет `/me`, публичный профиль `/profile`
   был спрятан за незаметной ссылкой в самом низу кабинета.

## Ритм страницы (токены)

| Токен | Значение | Зачем |
|---|---|---|
| `--page-gutter` | 28px / 20px (≤768px) | Единый боковой отступ контента. Совпадает с шапкой и футером — всё стоит на одной вертикали. |
| `--page-top` | 48px | Первый блок страницы под шапкой (h1, «Назад», дашборд). |
| `--section-gap` | 80px | Расстояние между крупными секциями внутри страницы. |
| `--page-bottom` | 120px | Последний блок до футера. |
| `--heading-gap` | 24px | Заголовок секции ↔ её контент. |

**Правило:** секция страницы задаёт padding **шорткатом целиком**
(`padding: var(--page-top) var(--page-gutter) 0;`), а не отдельными сторонами —
иначе всплывает унаследованный низ от глобального правила `section`.
Глобальный дефолт `section` теперь `var(--space-lg) var(--page-gutter)`.

Мобильный gutter переключается один раз через `:root { --page-gutter: 20px }`
в медиа-запросе — никаких точечных `padding-left: 20px` по страницам.

## Контейнеры

- `--container-max: 1200px` — маркетинговые страницы (index, about, сетки витрины).
- `--content-col: 860px` — «app»-страницы: профиль, кабинет, проект, формы.
  Все блоки одной страницы живут в **одном** контейнере — не смешивать.

## Типографика

- Размеры — только через `--text-*`. Хардкод px запрещён, кроме:
  - 13px — служебный размер mono-подписей (лейблы, футер-мета);
  - 15px — размер текста в инпутах и кнопках (`.field input`, `.btn-secondary`);
  - декоративные clamp()-заголовки hero.
- Дробных размеров (14.5px и т.п.) не существует.
- Иерархия app-страницы: h1 = `--text-4xl`, h2 секции = `--text-2xl` + `margin-bottom: var(--heading-gap)`.

## Радиусы

Только токены: `--radius-xs` (6px, чипы/бейджи) → `--radius-sm` (10px) →
`--radius-md` (16px, инпуты) → `--radius-lg` (22px, карточки) →
`--radius-xl` (30px, крупные карты/формы) → `--radius-full` (пилюли).
4px допустим только для декоративных полосок-скелетонов.

## Логика профиля

- Имя в шапке → `profile.html?id=<uid>` (публичный профиль, «как меня видят»).
- На своём профиле — пилюля «Редактировать профиль» → `me.html`.
- `/me` (кабинет) — форма + «Твои проекты» со статусами; под h1 — пилюля
  «Мой публичный профиль».
- Роли не дублируются: `/profile` — витрина, `/me` — редактор.

## Чек-лист для новых страниц

1. Первый блок: `padding: var(--page-top) var(--page-gutter) 0`.
2. Секции дальше: `padding: var(--section-gap) var(--page-gutter) 0`, последняя — низ `var(--page-bottom)`.
3. Один контейнер на страницу (`--content-col` или `--container-max`) + `margin: 0 auto`.
4. Цвета/размеры/радиусы — только токены.
5. `display:flex|grid` + `hidden` → явный гард `.класс[hidden] { display: none; }` (см. CLAUDE.md).
