## A. Единая ширина контента (главное)

**Сейчас** ширина контентной колонки прыгает между страницами:

| Страница | Селекторы | Ширина сейчас |
|---|---|---|
| шапка/футер | `.header-inner` / `.footer-inner` | 1200 + паддинг 28px |
| главная | `section` | 1200 + паддинг `--space-md` (80px) |
| `projects.html` | `.projects-hero h1` / `.hero-subtitle` | 900 / 600 |
| `project.html` | `.pd-back`, `.pd-hero`, `.pd-loading`, обсуждение | 860, центр |
| `submit.html` | `.submit-gate`, `.submit-form-wrap`, `.submit-success` | **560**, центр |
| `profile.html` | `.pf-hero` | **560**, центр |
| `admin.html` | `.adm-dashboard` | полная сетка section |

**Стандарт (зафиксирован Opus, не обсуждается в рамках задачи):**

1. **Сетка страницы** одна на всех — `section` с `max-width: var(--container-max)`
   и паддингами как на главной. Витрины/списки на полную сетку: `projects.html`
   и `admin.html` уже так — не трогать, только убедиться.
2. **Контент-колонка** для читаемого контента и форм — новый токен в
   `css/tokens.css` (`:root`): `--content-col: 860px`. Перевести на него:
   - `submit.html`: `.submit-gate`, `.submit-form-wrap`, `.submit-success`
     (560 → токен). Это решает «супер узкое поле описания».
   - `project.html`: все `.pd-*`-обёртки и блок обсуждения (сейчас 860
     хардкодом в нескольких местах — `grep -n "860" styles.css`) → токен.
   - `profile.html`: `.pf-hero`, `.pf-loading`, `.pf-not-found` (560 → токен).
     `.pf-projects` (сетка карточек) остаётся на полной сетке.
   - `.projects-hero h1` (900) → токен, чтобы заголовок витрины начинался и
     переносился в той же колонке. Лид `.hero-subtitle` (600) не трогать —
     это типографика, не контейнер.
3. Формы и текст на полные 1200px **не растягивать** — строка 140+ символов
   нечитаема. Требование «ровно как шапка» осознанно не выполняем; цель —
   чтобы колонка не прыгала между страницами.

Главная (`index.html`) при этом визуально меняться не должна.

## B. Textarea растёт под текст

**Сейчас:** `.submit-form textarea { min-height: 120px; resize: vertical }` —
длинное описание в edit-режиме приходится листать в маленьком окне.

- Хелпер в `js/util.js`: `autoGrowTextarea(el)` — на `input` ставит
  `el.style.height = 'auto'`, затем `el.style.height = el.scrollHeight + 'px'`.
  Вызвать и сразу при инициализации (edit-режим префилит значение до того).
- Подключить: textarea описания в `submit.html` (`js/submit.js`, в т.ч. после
  префилла edit-режима) и textarea коммента на `project.html` (`js/project.js`).
- Потолок: `max-height: 60vh` в CSS + `overflow-y: auto` — после потолка
  внутренний скролл. `resize: vertical` оставить. Начальная высота как сейчас.

## C. «← Все проекты» → «← Назад»

**Сейчас** на `project.html` и `profile.html` ссылка `.back-link`
«Все проекты» со стрелкой влево — читается как «Назад», но всегда ведёт на
витрину. Пришёл с главной → уносит не туда.

- Текст → «Назад» (строка в `js/i18n/ru.js`, напр. `nav.back`).
- Поведение: если пришли изнутри сайта — `history.back()`; иначе (прямая
  ссылка, внешний переход) — обычный переход на `projects.html`:

```js
backLink.addEventListener('click', (event) => {
  const cameFromSite = document.referrer
    && new URL(document.referrer).origin === window.location.origin
    && window.history.length > 1;
  if (cameFromSite) {
    event.preventDefault();
    window.history.back();
  }
  // иначе не мешаем ссылке вести на projects.html (href остаётся)
});
```

- `href="projects.html"` оставить — это и fallback, и поведение для новых
  вкладок/SEO. Отдельную кнопку «Все проекты» не добавлять — в шапке есть
  «Проекты».

## Ограничения

- Токены из `css/tokens.css`, новые размеры не хардкодить.
- Гард `[hidden] { display: none }` для любых новых классов с flex/grid,
  скрываемых через `hidden` (грабли проекта, 5 случаев — см. CLAUDE.md).
- Не ломать: edit-режим submit (T-EDIT), чипы стадии/запросов (T12), подсказку
  и чипы категории коммента (T13), скелетоны (T-UX4), sticky footer (T-UX5).

## Проверка (критерии приёмки)

Локальный сервер: `python3 -m http.server 8080` (не `file://`).

- Десктоп 1280×800: колонка контента идентична на `submit.html` (гейт и форма),
  `project.html`, `profile.html` — замерь `getBoundingClientRect()` левых/правых
  краёв, числа приведи в отчёте. Витрина и админка — полная сетка. Главная —
  без изменений.
- Textarea: edit-режим требует логина, поэтому проверь на обычной форме —
  вставь в textarea описания длинный текст (10+ абзацев) → поле выросло, весь
  текст виден без внутреннего скролла, после ~60vh — внутренний скролл.
  Авторост при префилле edit-режима проверь по коду (вызов хелпера после
  установки value). То же на форме коммента `project.html`.
- «Назад»: главная → карточка проекта → «Назад» = главная; витрина → проект →
  «Назад» = витрина; открытие `project.html?id=…` в новой вкладке → «Назад»
  ведёт на `projects.html`. То же для `profile.html`.
- Мобилка 375×812: без горизонтального скролла на всех правленых страницах.
- Консоль чистая; `npm run lint:js` и `npm run lint:css` — 0 ошибок.
