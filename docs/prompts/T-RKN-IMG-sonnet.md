# T-RKN-IMG — закрыть широкий `img-src https:` в CSP

Ты — Sonnet, исполнитель задачи We Designerz. Работай в текущем репозитории.
Общайся с пользователем по-русски.

## Зачем

Готовим подачу уведомления РКН по пути B: «трансграничная передача не
осуществляется» (`docs/19-rkn-submission.md`). CSP-meta всех страниц сейчас
разрешает `img-src ... https:` — браузер может грузить картинки с любого
https-хоста, включая иностранные. Надо сузить до явного allowlist.

## Инвентаризация уже сделана (Opus, 2026-07-18) — проверь и опирайся

- Статические картинки — только локальные `assets/...` (покрывается `'self'`).
- Динамические (обложки, галереи, аватары) — только наш Storage
  `https://api.wedesignerz.com/storage/...` (`cover_url` проверены живьём).
- `profiles.avatar_url` у всех текущих пользователей — `null`; UI загрузки
  аватара нет, поле заполняется только из `raw_user_meta_data` при signup
  (email-регистрация даёт null). Иностранных аватаров в проде нет.
- YouTube-превью (`i.ytimg.com`) в коде не используются — grep по `ytimg`/
  `youtube` в `js/` пуст. Эскалация из docs/19 не требуется.
- **Находка:** превью обложки/галереи в `js/submit.js` ставит
  `img.src = URL.createObjectURL(file)` — это `blob:`-URL, которого в
  `img-src` нет. Сейчас это молча спасает data:-fallback в `onerror`
  (см. комментарий в `submit.js` ~строка 405): blob блокируется CSP,
  onerror перечитывает файл как `data:`. Работает, но с CSP-violation в
  консоли и двойным чтением файла.

Перепроверь эти пункты grep'ом сам (не верь на слово): `ytimg|youtube`,
`avatar`, `createObjectURL`, `img.src` по `js/`.

## Что изменить

### 1. CSP-meta во всех 18 HTML (10 корень + 8 `projects/`)

Заменить в `img-src`:

- было: `img-src 'self' data: https:`
- стало: `img-src 'self' data: blob: https://api.wedesignerz.com`

Остальные директивы CSP не трогать (в т.ч. старый cloud-host в
`connect-src` — это rollback-граница, её убирают отдельной задачей).
Строка CSP должна остаться идентичной во всех 18 файлах.

`blob:` добавляем осознанно: после сужения `https:` превью в submit
перестало бы грузиться и вечно падало бы в fallback; с `blob:` основной
путь работает без CSP-violation. Fallback-код в `submit.js` не удалять —
он остаётся страховкой для браузерных конфигураций из комментария.

### 2. Проверить check-скрипты

`npm run check:csp` (см. `scripts/`) валидирует CSP-строки по всем HTML.
Если он утверждает старый `img-src` — обнови ожидание в скрипте в этом же
изменении. Логику других проверок не менять.

### 3. Отметить в docs/19

В `docs/19-rkn-submission.md` отметь чекбокс T-RKN-IMG выполненным с датой и
однострочным итогом (allowlist: self + data + blob + api.wedesignerz.com;
ytimg не используется; avatar_url — все null, валидация на свой Storage
остаётся в security follow-up docs/16). Владелец закоммитит вместе с кодом.
