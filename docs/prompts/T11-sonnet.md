## Что делаем

Только `projects.html` + его стили в `styles.css`. Главную (`index.html`) не трогать.

### A. Сжать hero страницы

Источник раздувания: `.projects-hero { min-height: calc(60vh - 80px);
justify-content: center }` (styles.css ~1263) — блок принудительно занимает 60%
экрана.
- Убрать `min-height`/центрирование, оставить компактные вертикальные отступы
  (порядка `--space-md` сверху, меньше снизу — на вкус, но hero должен стать
  «шапкой страницы», а не «экраном»).
- Состав блока сохранить: кикер, `h1`, подзаголовок, CTA «Добавить свой проект».
  Можно уплотнить (например, CTA в одну строку с подзаголовком на десктопе) —
  решай по месту, важен результат по критериям ниже.
- Анимации входа (fade-up) сохранить, `prefers-reduced-motion` уважать как везде.

### B. Компактные фильтры

- `.projects-filter` (`#filter`): убрать лишний вертикальный воздух между hero,
  фильтрами и сеткой. Чипы + сортировка на десктопе — одна строка (сейчас так и
  есть — не сломать), отступы ужать.
- На мобиле чипы могут скроллиться горизонтально в один ряд ИЛИ переноситься —
  выбери что чище, но фильтры не должны занимать пол-экрана.

### C. Результат: карточки видны раньше

- Десктоп 1280×800: первый ряд карточек виден без скролла (хотя бы верхняя треть
  карточек).
- Мобилка 375×812: верх первой карточки не глубже ~1.2 высоты экрана
  (`document.querySelector('.showcase-grid').getBoundingClientRect().top` при
  загрузке ≤ ~975px).

## Ограничения

- Токены из `css/tokens.css`, не хардкодить размеры/цвета, которые есть в токенах.
- Единую шапку (T-D5), логику фильтров/сортировки (`js/projects.js`) и разметку
  карточек (T-D3) не менять — только layout/spacing самой страницы.
- Не делать страницу «голой»: де-шаблонный характер (кикер-моно, ритм) сохранить,
  просто плотнее.
- Тексты не менять; если что-то захочется переформулировать — не в этой задаче.

## Проверка (критерии приёмки)

- Замеры из раздела C выполняются (проверь `getBoundingClientRect()` в консоли,
  напиши цифры в отчёте).
- Фильтры и сортировка работают как раньше (прокликай все чипы + обе сортировки).
- Пустое состояние витрины (когда фильтр ничего не находит) не разъехалось.
- Мобилка 375px: без горизонтального скролла.
- Консоль чистая; `npm run lint:js` / `npm run lint:css` — 0 ошибок.
