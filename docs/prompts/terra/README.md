# Промпты для Terra

Здесь лежат самодостаточные задания для Terra по инфраструктуре, безопасности
и cutover. Один файл = одна ограниченная задача с явными запретами, проверками
и форматом отчёта.

Порядок работы:

1. Тёма копирует содержимое одного промпта в новую сессию Terra.
2. Terra читает перечисленные в промпте файлы, сохраняет чужие
   незакоммиченные изменения и выполняет только заданный scope.
3. Terra не делает push/deploy и не меняет VPS/боевые данные без отдельного
   прямого разрешения.
4. Результат проверяют Тёма и «мозг» проекта; только после приёмки обновляется
   статус задачи в `docs/04-tasks-sonnet.md`.
