## A. Галерея не должна показывать «чёрный экран»

**Что случилось:** сразу после публикации 4 изображений автоплей галереи
(`js/project.js`, `buildGallery`/`startAutoplay`) переключился на картинку,
которая ещё не догрузилась (холодный CDN) — большая зона и миниатюра
отрисовались пустым тёмным прямоугольником. Файлы в Storage валидные,
после повторной загрузки страницы всё ок — дефект в отсутствии защиты.

Сделай:

1. **Автоплей пропускает недогруженные:** перед свапом проверяй готовность
   (`img.complete && img.naturalWidth > 0` у соответствующей миниатюры —
   большая зона использует тот же URL, файл общий). Если следующая не
   готова — перешагни на следующую готовую; если готовых кроме текущей
   нет — просто подожди следующего тика (не мигай).
2. **Ручной клик по недогруженной** — разрешён (юзер сам решил), но большая
   зона в момент ожидания не должна выглядеть «дырой»: пока
   `!coverImg.complete`, показывай на `.pd-cover` лёгкое состояние загрузки —
   например, класс с существующим shimmer-паттерном скелетонов (T-UX4,
   уже есть в styles.css) или пульс фона из токенов. Когда `load` сработал —
   снять. Событие `error` — тоже снять (не крутить вечно) и залогировать
   `console.warn`.
3. **Битые миниатюры:** на `error` у img миниатюры — скрой её кнопку
   (`hidden` + гард, если display flex) и выкинь индекс из ротации,
   чтобы автоплей не листал на мёртвый URL.

## B. Лимит дополнительных изображений 4 → 9

БД уже разрешает 9 (миграция — пререквизит; `schema.sql` синхронизирован
Opus'ом). На фронте:

- `js/submit.js`: `MAX_IMAGES` 4 → **9**.
- `js/i18n/ru.js`: тексты «до 4» → «до 9» (`submit.field.images*`,
  `submit.error.images_max` — проверь все вхождения «4» в этих строках).
- Вёрстка: 9 миниатюр в форме и 10 в галерее project.html должны
  переноситься на новые ряды (`flex-wrap`) и не давать горизонтальный
  скролл на 375px. Проверь оба места, поправь CSS при необходимости.

## Ограничения

- Токены CSS; гард `[hidden]{display:none}` для новых скрываемых flex/grid

## Проверка (критерии приёмки)

Локальный сервер: `python3 -m http.server 8080`.

- Эмуляция медленной сети (DevTools throttling или подмена одного URL на
  заведомо медленный/битый): автоплей не показывает пустую зону — пропускает
  недогруженную; клик по недогруженной показывает shimmer/пульс, не «дыру»;
  битый URL — миниатюра скрывается, ротация живёт.
- Живой проект «Стена» (4 изображения) — галерея работает как раньше.
- Форма: добавляются 9, десятая отбита с ошибкой, счётчик «9 / 9»;
  миниатюры переносятся рядами на 375px без h-скролла.
- Галерея с 10 картинками (эмуляция) — миниатюры в несколько рядов на
  мобилке, без h-скролла.
- Консоль чистая; `npm run lint:js` и `npm run lint:css` — 0 ошибок.
