## A. Манифест (короткая секция)

Тексты — из `docs/03-content.md`, раздел «Манифест»:
- Заголовок: «Ты всегда хотел что-то создать. Теперь — можешь.»
- Текст: «Это сообщество не для тех, кто и так умел кодить. Оно для идейных — кто годами
  носил в голове продукт, но не знал, как его собрать. Инструменты изменились. Барьер упал.
  Осталось придумать что-то по-настоящему полезное и сделать это. Мы сами такие — и делаем
  это каждый день.»
- Без CTA-кнопки — это смысловая пауза, не продажа. Компактная секция, крупная типографика.
- **Анти-шаблон:** не делать «центрированный заголовок + серый абзац». Обыграй в духе
  направления A (Электрический холст): например, моно-акценты JetBrains Mono, маркер `>`,
  асимметрия — посмотри, как решены hero-demo и how-terminal, и продолжи их язык.

## Ограничения

- Порядок секций главной: hero → showcase → how → **манифест** → **кто мы** → community → footer.
- Не трогать: шапку/навигацию (T-D5), hero, витрину, how-terminal, `#community`, footer,
  ленту участников (T-D4), auth-логику (T-UX1).
- Цвета/размеры — только из `css/tokens.css`. Шрифты: Onest + JetBrains Mono.
- Статические тексты — в HTML (как остальные секции главной), JS для этих блоков не нужен.
- Никаких выдуманных цифр и логотипов «клиентов, которых нет».

- На главной ровно один hire-CTA (в «Кто мы»), ведёт в личку `t.me/androsovart`;
  «Смотреть кейсы» ведёт на `about.html`.
- Тарифов/«Начать проект»/студийного оффера на главной нет (`grep -i 'тариф\|Начать проект'
  index.html` пуст).

- Секции не выглядят шаблонно (см. `06-design-direction.md`, «Анти-шаблон»).
- Консоль чистая; `npm run lint:js` / `npm run lint:css` — 0 ошибок.
