Проект We Designerz — статический сайт-клуб вайбкодеров, без сборки, GitHub Pages,
бэкенд Supabase. Прочитай: `CLAUDE.md`, `docs/08-workflow.md`,
`docs/10-membership.md` (весь — это основа задачи), `docs/03-content.md`
(«Вайб-рамка»).

Суть: членство должно иметь видимый рост. Этап 1 — два грейда, считаются
тривиально из существующих данных, без изменений БД:
- `v0 · новичок` — есть профиль (вступил в клуб);
- `v1 · строитель` — есть ≥ 1 опубликованный проект.

Метафора версий — родная для вайбкодинга: участник «релизит» сам себя.
Названия — черновик, финалит Тёма (см. таблицу шкалы в `10-membership.md`).

## A. Модуль грейда (`js/grade.js`)

- `computeGrade({ publishedCount })` → `'v0' | 'v1'`. Чистая функция, готовая к
  расширению (v2+ на этапе 2 — не реализовывать, но не зашивать логику намертво
  в рендер).
- Подписи в `js/i18n/ru.js`: `grade.v0` «v0 · новичок», `grade.v1` «v1 · строитель»
  + короткие пояснения для тултипа/подписи: v0 — «вступил — уже не смотрит со
  стороны», v1 — «зарелизил первый проект» (черновик, тон вайб-рамки).
- Рендер-хелпер бейджа: моно-шрифт (JetBrains Mono), тихий стиль — уровня меток
  `✦`; это знак чести, не кричащая плашка. Токены из `css/tokens.css`.

## B. Где показывается (этап 1 — только два места)

- `profile.html`: бейдж грейда рядом с именем в hero профиля. Считается из
  количества published-проектов участника (данные уже загружаются для сетки
  проектов — не делай лишний запрос, переиспользуй).
- `me.html`: тот же бейдж + одна строка «до v1: покажи первый проект» для v0
  (мотивация следующего шага; для v1 строки нет — v2 ещё не существует).
- Больше НИКУДА (карточки витрины, комменты, member-grid — не трогать: сначала
  смотрим, как читается в профиле).

## C. Честность

- Грейд вычисляется на клиенте из публичных данных — подделать в UI можно только
  себе локально, витрине это не вредит. В БД ничего не пишем (этап 2 решит,
  нужна ли колонка). Зафиксируй это комментарием в `js/grade.js`.

## Ограничения

- Строки — `js/i18n/ru.js`. Токены CSS. Гард `[hidden]` для новых flex/grid.
- Не ломать: profile.html (T-D4/T14), me.html (T14), скелетоны.
- Никаких «прогресс-баров геймификации» — сдержанно, в эстетике клуба.

## Проверка (критерии приёмки)

- Участник без проектов: на profile.html и me.html бейдж «v0 · новичок»,
  в me.html строка «до v1…».
- Участник с published-проектом: «v1 · строитель», мотивационной строки нет.
- Pending/rejected проекты НЕ дают v1 (только published).
- Бейдж не ломает вёрстку hero профиля на мобилке 375px.
- Консоль чистая; линтеры 0.
