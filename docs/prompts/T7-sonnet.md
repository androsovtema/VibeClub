## A. Гейт доступа

- На загрузке: получить сессию (как в других модулях через `js/supabase.js` / `js/auth.js`),
  затем свой `profiles.role`.
- Не залогинен или `role !== 'admin'` → показать честный экран «Доступа нет» со ссылкой
  на главную (без редиректа-мигалки; контент админки не рендерить).
- `<meta name="robots" content="noindex, nofollow">` в `<head>`.
- Смена auth-состояния (logout) — админка сменяется экраном «Доступа нет».

## B. Модерация проектов

Три вкладки (или сегмент-переключатель) — «На модерации» / «Опубликованные» / «Отклонённые»:

- **На модерации** (`status = 'pending'`, админ видит всё по RLS): карточка — обложка,
  название, автор (имя из `profiles`), описание, ссылка на проект (`target="_blank"`),
  теги/инструменты, дата. Кнопки:
  - «Опубликовать» → `update projects set status='published'`;
  - «Отклонить» → `status='rejected'` с инлайн-подтверждением (без `alert/confirm`).
- **Опубликованные**: тот же список + тумблер **`is_core`** («Флагман We Designerz» —
  звёздочка ✦ в витрине) и кнопка «Снять с публикации» (→ `pending`).
- **Отклонённые**: список + «Вернуть на модерацию» (→ `pending`).
- После действия карточка живо переезжает между вкладками без перезагрузки; счётчики
  в заголовках вкладок обновляются.
- Пустые состояния честные («Пока пусто»).

## C. Модерация комментариев

Отдельная секция/вкладка «Комментарии»: последние N (например 50) комментариев со
`status='published'` по всем проектам (join: автор + название проекта, ссылка на
`project.html?id=…`). У каждого — кнопка «Скрыть» → `update comments set status='hidden'`
(инлайн-подтверждение). Скрытый исчезает из списка и из ленты проекта (RLS/фильтры уже
показывают только `published` — не ломай).

## D. Витрина после публикации

Ничего дополнительно делать не нужно (витрина уже читает `status='published'`), но
проверь цепочку: pending-проект → «Опубликовать» → появился на `projects.html` и главной.

## Ограничения

- Никакого `service_role` — обычный клиент, права даёт RLS. Если RLS вернул ошибку —
  показать её честно (тост/инлайн), не глотать.
- Дизайн — направление A, токены из `css/tokens.css`, Onest + JetBrains Mono, шапка/футер
  как на остальных страницах (T-D5), стиль карточек можно взять от витрины/профиля.
- Тексты — в `js/i18n/ru.js`, пользовательские данные — `textContent`/`escapeHtml`.
- `admin.html` в `sitemap.xml` НЕ добавлять.

## Проверка (критерии приёмки)

- Гость и обычный участник видят «Доступа нет»; в Network при этом нет данных чужих
  pending-проектов (RLS отдаёт пусто).
- Под админом: pending-проект публикуется и появляется в витрине; отклонение, возврат,
  `is_core`-тумблер и скрытие коммента работают живо, без перезагрузки.
- Мобилка 375px без горизонтального скролла; консоль чистая;
  `npm run lint:js` / `npm run lint:css` — 0 ошибок.
