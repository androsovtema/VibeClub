## A. Клиентская оптимизация изображений

Сейчас `js/submit.js` грузит файл в Storage как есть (лимит 3 МБ). Сборки и
сервера нет — оптимизируем на клиенте через canvas.

Новый модуль `js/image.js`:

```js
export async function optimizeImage(file, { maxSide = 1600, quality = 0.82 } = {})
// → { blob, ext } // ext: 'webp' | 'jpg'
```

- **EXIF-ориентация обязательна:** `createImageBitmap(file,
  { imageOrientation: 'from-image' })` — иначе вертикальные фото с телефона
  лягут боком.
- Ресайз: если большая сторона > `maxSide` — уменьшить пропорционально
  (canvas). Меньше — не увеличивать.
- Кодирование: `canvas.toBlob(cb, 'image/webp', quality)`. Если браузер не
  умеет webp-энкод (старый Safari: blob null или `blob.type !== 'image/webp'`)
  — fallback `image/jpeg`, quality 0.85.
- **Страховка:** если результат больше исходника (исходник уже мелкий webp) —
  вернуть исходник как есть (`ext` по типу файла).
- PNG с прозрачностью: webp сохраняет альфу; в jpeg-fallback зальётся чёрным —
  перед drawImage залей канвас цветом фона `#0a0a0f` (близко к фону сайта).

В `js/submit.js`: входной лимит `MAX_COVER_BYTES` 3 МБ → **10 МБ** (текст
ошибки в i18n обнови), перед upload звать `optimizeImage`, расширение файла в
Storage-пути — из результата (`.webp`/`.jpg`). ACCEPTED-типы не расширять.

## B. До 5 изображений в submit

БД: `projects.images text[]` (до 4 доп. изображений, check в БД; обложка —
отдельно в `cover_url`, итого до 5).

- `submit.html/js`: под текущим выбором обложки — блок «Ещё изображения
  (до 4)»: тот же стилизованный выбор файла, превью-миниатюры добавленных с
  кнопкой × удаления. Каждый файл — через `optimizeImage`, тот же валидатор
  типа/размера. Обложка обязательна как была; дополнительные — опциональны.
- Insert: upload всех доп. файлов в `covers/<uid>/` → массив публичных URL в
  `images`. Если часть загрузок упала — честная ошибка, не полусохранённый
  проект (загружай доп. файлы ДО insert, как сейчас с обложкой).
- Edit-режим (`submit.html?id=…`): префилл миниатюр из `images`; удаление
  миниатюры и добавление новых; при сохранении удалённые старые файлы убрать
  из Storage (`supabase.storage.from('covers').remove([path])`, политика
  `covers_delete_own` есть; путь достань из публичного URL). Ошибку remove
  не показывать пользователю (мусор в Storage хуже, чем сломанный флоу, —
  залогируй в console.warn и продолжай).
- Порядок в массиве = порядок добавления. Перестановку не делаем (не раздувай).

## C. Галерея на project.html

Сейчас — одна обложка (`.pd-cover`). Галерея = `[cover_url, ...images]`
(валидные http-URL, мусор отфильтруй через `isHttpUrl`).

- Если картинка одна — всё ровно как сейчас, никаких новых элементов.
- Если больше: под большой зоной — горизонтальный ряд миниатюр (все картинки,
  включая обложку). Клик по миниатюре — большая зона показывает её (смена src,
  можно с лёгким fade). Активная миниатюра подсвечена (`aria-current` или
  `aria-pressed` + видимая обводка). Миниатюры — `button` с `img` внутри
  (клавиатура бесплатно), `alt` пустой (декоративные, название уже в h1).
- **Автоплей:** каждые 5 с — следующая по кругу. Останавливается **навсегда**
  (до перезагрузки страницы) при первом ручном переключении. Не запускается:
  при `prefers-reduced-motion: reduce`, при одной картинке. Бонус-гигиена:
  `clearInterval` не забудь и при уходе со страницы ничего не делать не надо
  (SPA нет).
- Высота большой зоны стабильна (текущий `aspect-ratio` 16/9, `object-fit`),
  миниатюры не ломают колонку `--content-col` и мобилку.

## Ограничения

- Токены CSS; XSS-гигиена: URL из БД только через проверку `isHttpUrl`,
  никаких innerHTML с данными; гард `[hidden]{display:none}` для новых
  flex/grid классов, скрываемых через `hidden` (грабли проекта, 5 случаев).
- Не ломать: T-EDIT (edit-режим), T12-чипы, скелетоны (T-UX4), колонку
  `--content-col` (T-UX6), карточки витрины (используют только `coverUrl`).
- RLS/политики не менять. `service_role` не существует для тебя.

## Проверка (критерии приёмки)

Локальный сервер: `python3 -m http.server 8080` (не `file://`).

- **A:** юнит-прогон в консоли: `optimizeImage` на большом jpeg (сгенерируй
  canvas-ом файл ~3000px) → blob webp, размеры ≤ 1600, вес кратно меньше.
  Вертикальная EXIF-картинка — проверь по коду наличие `imageOrientation`.
- **B:** форма — добавление 4 миниатюр, пятая не добавляется (счётчик/ошибка),
  удаление работает; в edit-режиме префилл (эмуляцией, без логина) и логика
  remove по коду. Реальный e2e upload — за Тёмой.
- **C:** страница проекта с `images` (можно локальной эмуляцией данных):
  миниатюры рендерятся, клик свапает, активная подсвечена, автоплей листает
  каждые 5 с и умирает после ручного клика; проект с одной обложкой — ни
  миниатюр, ни таймера (проверь отсутствие setInterval). Reduced-motion —
  автоплея нет.
- Мобилка 375×812 без горизонтального скролла (submit и project).
- Консоль чистая; `npm run lint:js` и `npm run lint:css` — 0 ошибок.
