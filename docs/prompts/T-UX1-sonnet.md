## A. Интерфейс различается: гость vs участник (главный пункт)

Сейчас залогиненному всюду показывается «Вступить», клик открывает модалку логина —
бессмыслица. Правило: **все `[data-join]`-CTA становятся auth-aware**.

- Новый маленький модуль/хелпер (например, в `js/app.js`, там уже есть `onAuthChange` +
  `renderHeaderAuth`): при смене auth-состояния прозрачно переключать CTA:
  - **Гость:** как сейчас — «Вступить» → модалка логина.
  - **Участник:** та же кнопка превращается в **«Добавить проект»** → `submit.html`
    (это главное действие участника). Касается: hero главной («Вступить в сообщество»),
    CTA-блока `#community` внизу главной, и любых других `data-join` вне шапки.
  - Гейты на `submit.html` (без логина) и в обсуждении — НЕ трогать, они и должны
    звать логин.
- Шапка залогиненного: **имя пользователя — ссылка на его `profile.html?id={uid}`**
  (сейчас просто текст). Рядом «Выйти» как есть.
- Реализация должна переживать порядок загрузки: `data-join`-обработчик в `app.js`
  делегированный — переключай поведение по текущему состоянию, а не перевешивай
  слушатели.

## B. Поле «Ссылка на проект» — авто-https (submit.html)

Пользователь пишет `ourwall.ru` — получает ошибку «должна начинаться с http://». Плохо.
- В `js/submit.js`: при blur поля и перед валидацией — если значение непустое и НЕ
  начинается с `http://` / `https://`, автоматически подставить `https://` в начало
  (видимо для пользователя — значение поля обновляется). Уже есть протокол — не трогать.
- Ошибка «должна начинаться с http://» остаётся только для совсем мусора
  (пробелы, кириллица без домена — то, что не парсится `new URL` даже с https://).
- То же самое поле `website` в профиле, если/когда появится форма редактирования — не
  в этой задаче, просто вынеси хелпер `normalizeHttpUrl` в `js/util.js`.

## C. Обсуждение на `project.html` — паттерн как у людей

1. **Форма ввода — НАД лентой комментов** (YouTube-паттерн): заголовок «Обсуждение» →
   форма (или гейт «Войди…») → лента. Сейчас форма под лентой — при большом числе
   комментов до неё не долистать.
2. **Сортировка ленты: новые сверху** (`created_at desc`) — согласуется с формой сверху:
   отправил → сразу видишь свой коммент под формой.
3. **Свой коммент можно удалить и отредактировать** (RLS уже разрешает: политики
   `comments_update_own_or_admin` / `comments_delete_own_or_admin`):
   - у своих комментов — маленькие кнопки «изменить» / «удалить» (иконки/текст, ненавязчиво);
   - «удалить» — инлайн-подтверждение («Удалить? Да / Отмена»), БЕЗ `alert/confirm`;
     после удаления коммент исчезает из ленты;
   - «изменить» — тело коммента заменяется на textarea с текущим текстом + «Сохранить» /
     «Отмена»; сохранение — `update` по id, в ленте обновляется без перезагрузки;
   - определение «свой»: `comment.author_id === currentUser.id` (добавь `author_id`
     в select комментов);
   - смена auth-состояния (выход) — кнопки исчезают.
4. Счётчик комментов на карточках витрины (`comments(count)`) продолжает считать только
   published — не ломай.

## D. Layout `project.html` — выравнивание и дыра

Замерено на ревью:
- Разрыв между верхней частью и «Обсуждением» ~300px
  (`.pd-hero` padding-bottom 180px + секция обсуждения padding-top 120px). Сжать до
  разумного (~64–80px суммарно, на вкус, но чтобы обсуждение было видно без «потерялся»).
- Левые края не совпадают: hero и обсуждение имеют разные max-width (860 vs 720) и
  разные отступы (замер: left 11px vs 81px). Выровнять: **одна контентная колонка**
  (одинаковые max-width и горизонтальные отступы) для верхней части и обсуждения.
- Проверь то же на мобиле (375px): единый левый край, без гигантских пустот.
- Заодно глянь, откуда пустая зона внутри hero при отсутствующем скриншоте на десктопе
  (по скрину Тёмы — большая чёрная область). Если это колонка обложки — при
  плейсхолдере она не должна раздувать блок.

## Проверка (критерии приёмки)
- Гость: «Вступить» → модалка. Участник: та же кнопка = «Добавить проект» → `submit.html`;
  в шапке имя → свой профиль. Переключение живое (логин/логаут без перезагрузки).
- `ourwall.ru` в поле ссылки → после blur само становится `https://ourwall.ru`, форма
  уходит без ошибки. `https://ourwall.ru` — не задваивается.
- Обсуждение: форма сверху, новые комменты сверху; свой коммент можно изменить и
  удалить (с инлайн-подтверждением), чужой — нельзя (кнопок нет). Проверь под своей
  учёткой из БД.
- `project.html`: левые края верхней части и обсуждения совпадают (проверь
  `getBoundingClientRect().left`), разрыв между блоками не более ~80px.
- Мобилка 375px — ничего не разъехалось.
- Консоль чистая; `npm run lint:js` / `npm run lint:css` — 0 ошибок.
