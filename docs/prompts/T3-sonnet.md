Проект We Designerz — статический сайт-сообщество вайбкодеров, без сборки, GitHub Pages,
бэкенд Supabase. Прочитай перед стартом: `CLAUDE.md`, `docs/08-workflow.md`,
`docs/01-architecture.md` (схема БД, RLS, Storage), `docs/03-content.md` (раздел
«Форма /submit.html»), `docs/06-design-direction.md` (поля форм, анти-шаблон),
`docs/04-tasks-sonnet.md` (задача **T3**). Уже готово: auth (T1: `js/auth.js`,
`js/ui/authModal.js`), витрина (T2: `js/projects.js`, `js/showcase.js`), общие утилиты
(`js/util.js`). Не делай шаблонно.

## Цель
Страница `submit.html` + `js/submit.js`: залогиненный участник добавляет проект —
тот падает в `projects` со `status='pending'` (на витрине не виден до модерации).

## A. Страница `submit.html`

- Шапка и футер — как на `projects.html` (nav + `data-header-auth`, подключить
  `js/app.js`). `<title>` «Покажи свой проект — We Designerz», meta по тону
  `03-content.md`.
- Заголовок: «Покажи свой проект».
- **Гейт без логина** (текст из `03-content.md`): «Чтобы добавить проект, войди — это
  быстро.» + кнопка, открывающая существующую модалку (`openAuthModal`). Google-кнопки
  НЕТ (Google-логин не делаем, см. T1) — только модалка (почта+пароль / магик-линк).
  Форма при этом скрыта.
- После логина (см. `onAuthChange` / `setAuthSuccessHandler`) гейт сменяется формой
  без перезагрузки страницы.

## B. Форма (все строки — через `js/i18n/ru.js`)

Поля:
1. **Название** — text, обязательное, максимум ~80 символов.
2. **Что это и как сделано** — textarea, обязательное. Плейсхолдер подтолкни к формату
   «что за продукт + как вайбкодил» (каким ИИ, что было сложно).
3. **Ссылка на проект** — url, обязательное, валидация формата (http/https).
4. **Обложка** — file, опциональное (в БД `cover_url` nullable; без обложки карточка
   получает градиент-плейсхолдер из T2). Валидация: jpeg/png/webp, ≤ 3 МБ. Превью
   выбранной картинки до отправки.
5. **Теги** — чипы-мультивыбор из тех же категорий, что на витрине: `prod`
   (Продуктивность), `biz` (Бизнес), `game` (Игры), `home` (Личное), `art` (Творчество).
   Используй `CATEGORY_LABELS` из `js/projects.js`. Минимум 1.
6. **Инструменты ИИ** — быстрые чипы (Claude, ChatGPT, Cursor, v0, Lovable, Bolt) +
   поле «свой вариант». Минимум 1. В БД — массив `tools`.

Кнопка: «Отправить на модерацию».

## C. Отправка — `js/submit.js`

1. Если есть обложка → upload в Storage bucket `covers`, путь строго
   `${user.id}/${crypto.randomUUID()}.${ext}` — RLS пускает запись только в папку,
   где первый сегмент = uid. Затем `getPublicUrl`.
2. `insert` в `projects`: `author_id = user.id`, `title`, `description`, `project_url`,
   `cover_url` (или null), `tags`, `tools`, `status: 'pending'`. Ничего кроме `anon`-ключа;
   RLS сам не даст вставить `published` или чужой `author_id` — не пытайся обходить.
3. Успех: форма сменяется блоком «Готово! Проверим и опубликуем. Загляни в чат, пока
   ждёшь.» + ссылка на Telegram-чат (возьми существующую ссылку t.me из футера).
4. Ошибки (upload/insert/сеть) — человеческим текстом возле кнопки, не `alert()`.
   Кнопка на время отправки блокируется («Отправляем…»), двойной сабмит невозможен.

## Безопасность (обязательно)
- Весь пользовательский текст при выводе (превью, сообщения об ошибках с данными) —
  через `textContent`/`createElement` или `escapeHtml` из `js/util.js`.
- Никакого `service_role`. Только ESM + ванильный JS, без либ.

## Стиль
- Токены из `css/tokens.css`, никакого хардкода цветов/размеров, где есть токен.
- Поля форм — в стиле модалки логина (посмотри её CSS), тёмная тема, моно-акценты.
- Не «типовая форма»: см. `06-design-direction.md`. Мобильная вёрстка обязательна.

## Проверка (критерии приёмки)
Локально (`python3 -m http.server 8080`, порт может быть занят — возьми другой):
- Незалогиненный видит гейт, не форму. Кнопка гейта открывает модалку.
- После логина форма появляется без перезагрузки.
- Отправка без обязательных полей / с кривым URL / с не-картинкой / с файлом > 3 МБ —
  понятные ошибки, запрос не уходит.
- Успешная отправка: проект в БД со `status='pending'` (проверь в table editor или
  select'ом), на витрине НЕ виден, обложка лежит в `covers/<uid>/...`, `cover_url`
  корректный.
- Экран успеха показывается; повторная отправка того же сабмита невозможна.
- Консоль чистая. `npm run lint:js`, `npm run lint:css` — 0 ошибок.
