## Проблема

Блок «we-designerz · превью» в hero главной (`index.html`, `.hero-demo`;
логика — `js/main.js`, секция `HERO DEMO`, ~строка 245): печатаются 3 промпта
по кругу, но выдача — статичный DOM «Кофейня „Зерно“» с одними и теми же
карточками. Промпт «трекер привычек» → выдача «кофейня». Выглядит дёшево,
убивает магию живой генерации.

## Что сделать

### A. Данные: пары «промпт → выдача»

Массив из **12–16 пар** в `js/main.js` рядом с логикой hero demo
(`main.js` не ES-модуль и не может импортировать i18n — контент-фикстура
в самом файле, это осознанно; оставь комментарий). Формат каждой пары:

```js
{ prompt: 'приложение-трекер привычек', icon: '✅', title: 'Привыкун',
  media: ['oklch(0.4 0.12 150 / 50%)', 'oklch(0.45 0.1 200 / 50%)'] }
```

- `prompt` — как реально пишет человек: разговорный, без канцелярита,
  нижний регистр («сделай лендинг кофейни с меню», «портфолио фотографа
  с галереей», «трекер расходов для семьи», «сайт для барбершопа с записью»…).
  Существующие 3 промпта сохрани как часть набора.
- `title` — название сервиса, которое «сгенерировалось»: живое, в духе
  вайб-рамки (`docs/03-content.md`), коротко (≤ 18 символов, чтобы не
  ломать высоту блока): «Кофейня „Зерно“», «Привыкун», «Кадр», «Стрижка №1»…
- `icon` — **эмодзи** (☕ ✅ 📷 💈 …), НЕ внешняя иконочная библиотека:
  внешних запросов и зависимостей не добавляем (решение Opus, зафиксировано
  в задаче). Эмодзи рендерить отдельным `<span aria-hidden="true">` перед
  названием.
- `media` — два цвета для `.hero-demo-card-media` (сейчас захардкожены
  inline в `index.html`), чтобы «скриншоты» выдачи различались тоном.
  Тона держи в существующей гамме (тёмные oklch с прозрачностью, как сейчас).

### B. Рендер выдачи под пару

- Перед сборкой блоков (`data-asm`) подставлять `icon + title` в
  `.hero-demo-block-title` и цвета в `.hero-demo-card-media` текущей пары.
  Текст — через `textContent` (иконку — отдельным span), не innerHTML-строкой.
- Смена содержимого — в момент, когда блоки скрыты (opacity 0), чтобы не было
  видимой подмены.
- **Высота блока стабильна:** название в одну строку, эмодзи не увеличивает
  line-height (проверь), карточки не меняют размеров. Вариации раскладки
  выдачи (например, 3 строки-списка вместо 2 карточек) — можно, но только
  если высота `.hero-demo` не прыгает между парами; если сомневаешься — не
  делай, иконка+название+цвета уже дают нужное разнообразие.

### C. Порядок показа

- При загрузке перемешай массив (Fisher–Yates) или стартуй со случайного
  индекса — повторный визит начинается с другой пары.
- `prefers-reduced-motion`: как сейчас — статичная одна пара (первая после
  перемешивания), без анимации, статус «✓ готово».

## Ограничения

- Токены/гамма из `css/tokens.css`; вёрстку `.hero-demo` радикально не
  перестраивать — это точечное оживление, не редизайн.
- Никаких внешних запросов (иконки, шрифты, библиотеки).
- Не ломать: тайминги и статусы («печатаю промпт…» → «● генерирую…» →
  «✓ готово»), скелет hero, живые счётчики (`js/stats.js`), порог их
  показа (T15).

## Проверка (критерии приёмки)

Локальный сервер: `python3 -m http.server 8080` (не `file://`).

- Прогони ≥ 3 полных цикла пар: выдача всегда соответствует промпту
  (иконка + название + цвета меняются), подмены на виду нет.
- Замерь высоту `.hero-demo` (`getBoundingClientRect().height`) на 3+ разных
  парах — числа идентичны (допуск ±1px), hero не «дышит».
- Перезагрузи страницу 2–3 раза — стартовая пара меняется.
- Эмуляция `prefers-reduced-motion: reduce` — статичная пара, всё видно.
- Мобилка 375×812: блок влезает, без горизонтального скролла.
- Консоль чистая; `npm run lint:js` и `npm run lint:css` — 0 ошибок.
