## B. `me.html` + `js/me.js` (новые файлы)

- Гейт: не залогинен → текст + кнопка `data-join` (тон вайб-рамки, «Вступи или
  войди»). Залогинен → кабинет.
- Редактирование своего профиля: `display_name`, `bio`, `telegram`, `website`
  (через `normalizeHttpUrl` из `js/util.js`), `skills` (ввод тегов: текстовое
  поле + Enter/кнопка «Добавить», чипы с крестиком), `open_to` (мультивыбор-чипы).
- Кнопка «Сохранить» → update `profiles` по своему `id` (RLS уже разрешает
  update own). Тост об успехе.
- Список **своих** проектов со статусами: `pending` «на модерации», `published`
  «в витрине», `rejected` «отклонён» (RLS уже отдаёт свои любого статуса).
  Статус — тихий моно-бейдж на карточке. Ссылка «Показать первый проект» →
  `submit.html`, если проектов нет.
- Ссылка «Мой публичный профиль» → `profile.html?id=<uid>`.
- Шапка/каркас страницы — как у `profile.html` (header, footer, скелетоны).

## Ограничения

- Строки — `js/i18n/ru.js`, тон — вайб-рамка (кабинет — тоже клуб: не «личный
  кабинет пользователя», а по-человечески).
- Чужой профиль редактировать нельзя: UI не даёт (me.html работает только со
  своим uid), RLS отбивает — проверь вручную запросом с чужим id (должен 42501/0 rows).
- `skills` — пользовательский ввод: рендер только `textContent`, обрезка длины,
  без HTML. XSS-тест: тег `<img src=x onerror=alert(1)>` в скилле — рендерится
  как текст.
- Токены CSS; гард `[hidden]`; не ломать T-UX1–T15 (auth-aware шапку, welcome).
