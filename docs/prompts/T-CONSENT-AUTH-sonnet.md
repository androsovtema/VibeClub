## Что изменить

### 1. Два независимых подтверждения

В signup-форме замени единый `data-consent` на два отдельных чекбокса. Оба
unchecked по умолчанию и оба нужны для регистрации:

1. processing consent:
   - текст: `Даю согласие на обработку моих персональных данных на условиях политики конфиденциальности.`;
   - слова `политики конфиденциальности` — ссылка только на `privacy.html`;
2. правила клуба:
   - текст: `Принимаю правила клуба.`;
   - слова `правила клуба` — ссылка только на `privacy.html#rules`.

Добавь понятные стабильные hooks, например `data-processing-consent` и
`data-rules-consent`. У каждого input должен быть собственный `id`, а у label —
точный `for`; не оставляй два безымянных вложенных label, если явные связи проще
проверить.

Строки вынеси в `js/i18n/ru.js`. Не собирай юридический текст из HTML внутри
бизнес-логики; сохрани принятый в проекте безопасный способ рендера ссылок.

### 2. Гейт регистрации

- Кнопка `data-signup-submit` включается только когда отмечены оба чекбокса.
- Изменение любого чекбокса пересчитывает состояние кнопки одной общей
  функцией, без дублирования логики.
- После loading/error кнопка восстанавливается по состоянию обоих чекбоксов.
- В submit-handler до `signUpEmailPassword()` явно проверь оба значения,
  несмотря на disabled-кнопку и `required`.
- Добавь отдельные i18n-ошибки:
  - `Подтверди согласие на обработку персональных данных.`;
  - `Прими правила клуба, чтобы вступить.`
- Не меняй `signUpEmailPassword()`: в metadata по-прежнему передаётся только
  текущая версия processing consent рядом с `display_name`.
- Signin, magic link, forgot/reset и confirm-view не должны измениться.

### 3. Стили и доступность

- Используй существующий `.auth-modal-consent`; допустимо превратить его в
  вертикальную группу с токенизированным gap.
- Оба чекбокса должны иметь кликабельный label, видимый keyboard focus и
  достаточный интервал между строками.
- Не добавляй новую карточку, иконки или иной визуальный язык.
- Проверь modal на 375 px без horizontal overflow.
- Если новый flex/grid-контейнер скрывается через `hidden`, добавь явный
  `.class[hidden] { display: none; }` guard.

### 4. Версия согласия

Поскольку меняется формулировка processing consent до live, синхронно замени:

`privacy-2026-07-15-v3` → `privacy-2026-07-16-v4`

только в:

- `js/consent.js`;
- `supabase/migrations/20260715091536_t_consent_01_user_consents.sql`;
- `supabase/schema.sql`.

Не создавай новую migration: существующая ещё не применялась. Не меняй SQL
кроме версии.

- прочие auth-потоки не сломаны;
- desktop/375 px, клавиатура, линтеры и version-check зелёные;
- production, VPS, deploy и git history не изменены.
