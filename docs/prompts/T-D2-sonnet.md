---

Проект We Designerz — статический сайт-сообщество вайбкодеров, без сборки, GitHub Pages,
бэкенд Supabase. Прочитай перед стартом: `CLAUDE.md`, `docs/08-workflow.md`,
`docs/06-design-direction.md` — раздел **«Анти-шаблон»** (это ТЗ данной задачи, читай
целиком), `docs/00-vision.md` (принцип честности), `docs/04-tasks-sonnet.md` (задача
**T-D2**). Уже готово: T1–T4 (auth, витрина, submit, страница проекта), YouTube-карточки.

## Цель
Главная (`index.html` + `styles.css` + при необходимости `main.js`) перестаёт выглядеть
как типовой AI-лендинг. Режим — **средняя де-шаблонизация**: скелет страницы сохраняем,
подачу меняем. Плюс убрать фейковые цифры (принцип честности).

## A. Честные цифры вместо фейка (обязательно, это прод-блокер)

В hero захардкожено «1 248 участников · 320 проектов» — выдумка. Заменить:
- Новый модуль `js/stats.js`: реальные счётчики из Supabase —
  `profiles` (`select('*', { count: 'exact', head: true })`) и
  `projects` со `status='published'` (тот же приём). RLS позволяет обе выборки анониму.
- Рендер в hero. Пока цифры маленькие — это нормально и честно; подача: моно-шрифт,
  формат «участников: 3 · проектов: 3 · опыта в коде: 0» (нулевая строка «0 строк опыта
  нужно» — оставить, она концептуальная, не фейк).
- Если запрос упал — блок цифр скрыть, не показывать нули с ошибкой.

## B. Кикер-под-копирку

`.section-label` сейчас над каждой секцией по схеме «кикер → H2 → лид». Убрать рефлекс:
- У «витрина сообщества» — заменить кикер на моно-фрагмент промпта (мотив терминала:
  `> покажи свежие проекты сообщества` с `>`-маркером и кареткой).
- У «как работает вайбкодинг» — убрать кикер совсем, вход через крупное одно слово или
  число (см. C).
- `community-cta` — оставить/убрать по вкусу, но чтобы на странице НЕ было двух секций
  с одинаковым паттерном входа подряд.

## C. Блок «как это работает» (`.how-strip`) — самый шаблонный, переделать

Уйти от «N шагов в ряд со стрелками». Направление (можешь предложить своё в этом духе):
терминальная рамка (точки-светофор, `>`-маркеры) — шаги как строки сессии вайбкодинга:
промпт → ответ ИИ → результат; номера шагов как крупная моно-цифра слева, асимметрично.
Мобильная вёрстка обязательна. `prefers-reduced-motion` уважать (анимации — только
прогрессив-энхансмент).

## D. Ритм и асимметрия

- Сломать одинаковые отступы между секциями: плотная витрина ↔ разреженное крупное
  утверждение (манифест) ↔ плотный блок.
- Хотя бы одна секция с явной асимметрией (сдвинутый заголовок / контент не по центру).
- ✦-глиф — сквозной: маркеры списков, разделители, footer уже есть. Не переборщи —
  2–3 новых применения достаточно.

## Что НЕ трогать
- Витрину (T2) и её карточки (T-D3) — они готовы и приняты.
- `hero`-терминал (живой промпт) — он уже хорош; можно лишь усилить связку
  «терминал → витрина» (напр., последняя строка терминала «толкает» к витрине).
- Тосты «X только что добавил проект» и живую ленту участников — это T-D4, не делай.
- Другие страницы — только главная (T-D5 разведёт Telegram-ссылки на остальных).

## Ограничения
- Токены `css/tokens.css`, ничего не хардкодить, где есть токен.
- Строки — `js/i18n/ru.js`, пользовательских данных тут нет, но дисциплина та же.
- Без новых библиотек. ESM + ванильный JS.
- Не ломать якоря `#showcase`, `#how`, `#community` (на них ссылается навигация).

## Проверка (критерии приёмки)
- В hero — реальные цифры из БД (сверь с table editor), при пустой сети блок скрыт.
- Нет двух секций подряд с одинаковым входом «кикер → H2 → лид».
- `.how-strip` в старом виде не существует; новый блок не «4 карточки со стрелками».
- Секции различаются плотностью/подачей; есть асимметрия.
- Мобилка (375px) — ничего не разъехалось; `prefers-reduced-motion` — без анимаций.
- Консоль чистая; `npm run lint:js` / `npm run lint:css` — 0 ошибок.
- Скриншоты «до/после» в отчёт (или опиши, что где изменилось, чтобы Тёма глянул).
