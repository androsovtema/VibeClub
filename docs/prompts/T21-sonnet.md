Проект We Designerz — статический сайт-клуб вайбкодеров, без сборки, GitHub Pages,
бэкенд Supabase (email-auth, защита на RLS). Прочитай: `CLAUDE.md`,

## Контекст: провальный e2e-прогон 2026-07-11

Первая живая регистрация с включённым Confirm Email провалилась по UX:

1. Ссылка из письма увела на 404 GitHub: `signUp` не передаёт `emailRedirectTo`,
   GoTrue средиректил на Site URL (был настроен без `/VibeClub/`). Site URL Тёма
   чинит в Dashboard, но код обязан не зависеть от него.
2. После сабмита формы регистрации показывается только строчка «Почти готово…»,
   а форма с активной кнопкой «Вступить в клуб» остаётся. Пользователь жмёт её
   повторно, думает что регистрируется заново — на деле Supabase молча
   возвращает 200 без письма (анти-enumeration для существующего юзера).
   Состояние непонятное, автосохранение пароля браузером не срабатывает.
3. Сессия с токенами улетела на 404-страницу и потерялась: пользователь
   подтвердил почту, но на сайте остался разлогиненным без объяснения.
4. Пути «забыл пароль» нет вообще.

Конфиг Supabase (Site URL, SMTP, шаблоны писем) — НЕ твоя зона, это делает Тёма
в Dashboard. Твоя зона — код.

## Выход

### 1. `emailRedirectTo` при регистрации

`js/auth.js`, `signUpEmailPassword`: добавить в `options`
`emailRedirectTo: window.location.origin + window.location.pathname`
(как уже сделано в `signInMagicLink`). Ссылка из письма должна возвращать
на ту же страницу, откуда регистрировались, и на localhost при локальной
разработке.

### 2. Состояние «проверь почту» вместо формы

`js/ui/authModal.js`, ветка `signup` без `data.session`:

- скрыть форму регистрации целиком, показать состояние подтверждения:
  заголовок, текст «Письмо ушло на **<email>**. Перейди по ссылке в письме —
  и ты в клубе» (email подставить реальный), моно-акцент в стиле модалки;
- кнопка «Отправить письмо ещё раз» → `supabase.auth.resend({ type: 'signup',
  email, options: { emailRedirectTo } })`, после клика — cooldown 60 сек
  с обратным отсчётом на кнопке (`disabled`), ошибка rate limit (429 /
  `over_email_send_rate_limit`) — человеко-читаемо: «Слишком часто. Подожди
  пару минут и попробуй снова»;
- ссылка «Ввёл не ту почту? Назад к форме» — возврат к форме регистрации;
- состояние живёт, пока модалка открыта; повторное открытие модалки —
  снова обычные вкладки входа/регистрации.

### 3. Детект «уже зарегистрирован»

При signup существующего подтверждённого юзера Supabase возвращает
обфусцированный ответ: `data.user` есть, `data.user.identities` — пустой
массив, сессии нет. Детектить этот случай и вместо «проверь почту» показывать:
«Эта почта уже в клубе. Войди — или восстанови пароль», с переключением на
вкладку входа (email предзаполнить). НЕ раскрывать больше, чем Supabase сам
отдаёт (не делать отдельных запросов «существует ли юзер»).

### 4. Возврат по ссылке из письма

`detectSessionInUrl` у supabase-js включён по умолчанию — после правильного
редиректа сессия подхватится сама. Добавить:

- на `SIGNED_IN`, пришедший из URL-токенов (первый auth-ивент при загрузке
  страницы с `#access_token` в хеше), показать тост «Почта подтверждена —
  ты в клубе ✦» (существующий механизм тостов);
- welcome-модалка новичка уже завязана на возраст профиля (T-UX7,
  `isFreshRegistration`) — проверь, что она срабатывает при таком входе,
  и что тост и welcome не дублируют друг друга (welcome приоритетнее);
- убедиться, что хеш с токенами не остаётся в адресной строке
  (supabase-js чистит сам — просто проверь).

### 5. «Забыл пароль»

- На вкладке входа — ссылка «Забыл пароль?» → поле email (можно переиспользовать
  введённый) → `supabase.auth.resetPasswordForEmail(email, { redirectTo:
  window.location.origin + window.location.pathname })` → то же состояние
  «письмо ушло» с resend-cooldown;
- по возврату Supabase шлёт ивент `PASSWORD_RECOVERY` в `onAuthStateChange` —
  открыть модалку в режиме «новый пароль» (одно поле + подтверждение кнопкой)
  → `supabase.auth.updateUser({ password })` → тост «Пароль обновлён», юзер
  залогинен;
- ошибки — человеко-читаемо через существующий `mapAuthError`.

### 6. `mapAuthError` — дополнить

Минимум: `invalid_credentials` («Неверная почта или пароль»),
`over_email_send_rate_limit` («Слишком часто…»), `email_not_confirmed`
(«Почта ещё не подтверждена — проверь письмо»). Все новые строки — через
`js/i18n/` как остальные тексты модалки, не хардкодом в JS.

## Чего НЕ делать

- Не трогать конфиг Supabase и не менять `js/config.js`.
- Не менять RLS, схему БД, триггеры.
- Не вводить OAuth-провайдеров.
- Ничего сверх задачи.

## Приёмка

- Регистрация на `localhost:8080` (Confirm Email включён): после сабмита —
  состояние «письмо ушло на <адрес>», кнопки «Вступить» не видно, resend
  работает с отсчётом; клик по ссылке в письме возвращает на localhost,
  юзер залогинен, тост/welcome показаны, хеш чистый.
- Повторный signup той же почтой — «Эта почта уже в клубе», вкладка входа.
- «Забыл пароль» — полный цикл до нового пароля и залогиненного состояния.
- Неверный пароль при входе — внятная ошибка по-русски.
- Мобилка 375px: все новые состояния модалки не ломаются.
- Линтеры 0 ошибок; `npm run security-check` зелёный.

Коммит один: `feat(T21): понятный путь подтверждения почты — redirect, resend,
детект повторной регистрации, восстановление пароля`.
