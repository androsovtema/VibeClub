## B. Дизайн-токены вместо хардкода (правки из T2)

В новых секциях `styles.css` (`.community-*`, `.showcase-*`, `.projects-filter-row`)
заменить сырые значения на токены из `css/tokens.css`, где есть прямой эквивалент:

- `font-size: 14px` → `var(--text-sm)`, `12px` → `var(--text-xs)`, `16px` → `var(--text-base)`
  и т.п. (11/13/15px — округлить до ближайшего токена, визуально проверить);
- `color: #ffffff` → `var(--color-text-primary)`;
- фоны бейджей вида `oklch(15% 0.02 282 / 60%)` → строить от токена, напр.
  `color-mix(in oklch, var(--color-bg) 60%, transparent)` (подобрать, чтобы визуально
  не изменилось);
- отступы `gap/padding` — на `--space-*`, где совпадает.

Градиенты-плейсхолдеры обложек в `js/projects.js` НЕ трогай — осознанное исключение
(декоративная палитра), добавь только комментарий об этом рядом с `COVER_GRADIENTS`.
