## Контекст

Целевые числа зафиксированы (`docs/00-vision.md`), но путь пользователя не
измеряется: нельзя понять, где течёт воронка (видео не приводит / регистрация
не проходит / форма тяжёлая). Решение Тёмы: **Umami Cloud** (cloud.umami.is) —
бесплатный тариф, без cookies (баннер согласия не нужен), кастомные события.
Полная постановка — `docs/04-tasks-sonnet.md`, раздел T19.

## Выход

### 1. Модуль `js/analytics.js` (новый) + конфиг

- В `js/config.js` — константа `UMAMI_WEBSITE_ID = ''` (id вставит Тёма после
  создания сайта в Umami; пустая строка = аналитика полностью выключена, ни
  скрипта, ни ошибок).
- `js/analytics.js` экспортирует `initAnalytics()` и `track(event, props)`:
  - `initAnalytics()`: если `UMAMI_WEBSITE_ID` пуст **или**
    `location.hostname` — `localhost`/`127.0.0.1`, скрипт НЕ грузим; в этом
    режиме `track` пишет `console.debug('[analytics]', event, props)` — так
    проверяем локально. Иначе — динамически создаём
    `<script defer src="https://cloud.umami.is/script.js">` с
    `data-website-id` из конфига и добавляем в `<head>`.
  - `track(event, props)`: обёртка над `window.umami?.track(...)` — если
    скрипт зарезал адблок или он ещё не загрузился, тихий no-op. Никаких
    исключений наружу, страница не падает никогда (приёмка).
  - Никакого PII в событиях: не слать email, user_id, тексты пользователя.
    Только имена событий и короткие enum-пропсы.
- Подключение `initAnalytics()` — один раз в общем `js/main.js`
  (как остальные общие модули), НЕ отдельным тегом в каждый HTML.

### 2. События воронки

Pageview и UTM-метки Umami собирает сам (авто-трекинг), «повторные визиты»
видны в дашборде как returning visitors — для них ничего не пишем. Кастомные
события (имена — snake_case, английские):

| Событие | Где | Пропсы |
|---|---|---|
| `join_click` | клики по кнопкам «Вступить» (все точки) | `place`: `nav` / `hero` / `gate` — по месту кнопки |
| `auth_open` | открытие auth-модалки | — |
| `auth_success` | успешный вход/регистрация | `kind`: `signup` / `login` / `magic` |
| `submit_start` | первое взаимодействие с формой проекта (один раз на загрузку страницы, например первый `input`) | — |
| `submit_sent` | успешный insert проекта | — |
| `publish` | админ опубликовал проект из админки | — |
| `comment_sent` | успешная отправка комментария | `first`: `true` только если в `localStorage` нет флага «уже комментировал» (после отправки флаг ставим) |

Точки интеграции ищи в существующих модулях (`authModal.js`, скрипт submit,
`project.js`, `admin.js`) — вызовы `track()` в местах, где уже есть обработка
успеха. Не дублируй логику, не меняй поведение существующего кода.

### 3. Документация (правки в docs/)

- `docs/05-launch.md`: шаблон UTM для ссылок из YouTube-описаний, например
  `?utm_source=youtube&utm_medium=video&utm_campaign=<слаг-видео>` + одна
  строка «как смотреть»: Umami сам разбирает UTM, дашборд → раздел UTM.
- `docs/09-growth-plan.md`: короткая таблица «метрика → порог → решение»
  (недельный ритуал): по одной строке на шаг воронки
  (визит → join_click → auth_success → submit_sent → publish → comment_sent),
  колонка «что меняем, если конверсия ниже». Черновые пороги придумай сам,
  пометь «пороги — черновик, финалит Тёма».

## Границы

- Никаких изменений БД/RLS — задача целиком клиентская.
- Не Google Analytics, не Метрика, никаких cookies и баннеров согласия.
- Не трогать вёрстку и стили (кроме, если нужно, ничего).
- Email-уведомления, серверные события — не в этой задаче.
- Упоминание счётчика на странице приватности — придёт в T20, здесь не делать.

## Приёмка

- Локальный прогон: `UMAMI_WEBSITE_ID` пуст → в консоли `console.debug`-события
  по всем точкам воронки (открыть модалку, войти, начать/отправить форму,
  коммент), реальных запросов к umami нет, ошибок нет.
- С заполненным тестовым id и хостом ≠ localhost (можно временно подменить
  гард) — в Network уходят POST к `cloud.umami.is`.
- Адблок-симуляция: заблокируй запрос скрипта в DevTools → страница работает,
  `track` молчит, консоль без ошибок (приёмка «graceful»).
- Ни одно событие не содержит PII.
- 375px не затронут (вёрстку не меняли), линтеры 0.
- В ответе — напоминание Тёме: завести аккаунт на `umami.is` (Cloud, free),
  добавить сайт → скопировать website id в `js/config.js` →
  `UMAMI_WEBSITE_ID`, задеплоить и проверить живые события в дашборде.
