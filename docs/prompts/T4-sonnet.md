Проект We Designerz — статический сайт-сообщество вайбкодеров, без сборки, GitHub Pages,
бэкенд Supabase. Прочитай перед стартом: `CLAUDE.md`, `docs/08-workflow.md`,
`docs/01-architecture.md` (схема: `comments`, `project_upvotes`, триггер пересчёта
`projects.upvotes`, RLS), `docs/03-content.md` (раздел «Карточка проекта /project.html»),
`docs/06-design-direction.md`, `docs/04-tasks-sonnet.md` (задача **T4**). Уже готово:
auth (T1), витрина и `fetchProjectById` в `js/projects.js` (T2), форма добавления (T3).
Карточки витрины уже ведут на `project.html?id={id}`. Не делай шаблонно.

## Цель
Страница `project.html?id=...` + `js/project.js`: полная карточка проекта, обсуждение,
апвоут. Всё из Supabase, только published-проекты (RLS это уже гарантирует).

## A. Страница `project.html`

- Шапка/футер — как на `submit.html` (nav + `data-header-auth`, `js/app.js`).
- Данные — `fetchProjectById(id)` из `js/projects.js` (id из `location.search`).
  При необходимости расширь select (например, добавь выборку комментариев отдельным
  запросом — см. B), но не ломай сигнатуру для витрины.
- Контент карточки:
  - обложка (или градиент-плейсхолдер как в витрине — переиспользуй логику из
    `js/projects.js`, вынеси хелпер, не копипасть);
  - заголовок, автор (аватар + имя), дата, бейджи инструментов (моно), теги
    (человекочитаемые из `CATEGORY_LABELS`), ✦ если `is_core`;
  - описание («что это и как сделано»);
  - заметная кнопка «Открыть проект →» на `project_url`
    (`target="_blank" rel="noopener noreferrer"`);
  - блок апвоута: `♥ {upvotes}` — см. C.
- **Плохой/чужой id** (нет в БД, не published, кривой uuid, нет `?id`): аккуратный
  экран «Проект не найден» + ссылка на витрину `projects.html`. Никаких сырых ошибок
  в UI. Title страницы — по проекту (`{title} — We Designerz`), при ошибке — «Проект
  не найден — We Designerz».
- Мета-теги: базовые статические (страница динамическая, краулеры без JS увидят
  статику — это ок для MVP). `robots: noindex` НЕ ставь.

## B. Обсуждение (`comments`)

- Заголовок секции: «Обсуждение» (строки — в `js/i18n/ru.js`).
- Лента: только `status='published'` комменты этого проекта (RLS и так отфильтрует,
  но `.eq` поставь явно), сортировка по `created_at` по возрастанию. Аватар + имя +
  дата + текст. Автор — join на `profiles` (внимание: указывай FK явно, как в T2 —
  `profiles!comments_author_id_fkey` — иначе может быть неоднозначность).
- Пусто: «Сообщений пока нет. Напиши первым.»
- Форма: залогиненный видит textarea + кнопку; незалогиненный — «Войди, чтобы
  оставить сообщение.» + кнопка `data-join` (модалка). После логина форма появляется
  без перезагрузки (`onAuthChange`).
- Отправка: insert `{ project_id, author_id: user.id, body }`. После успеха — коммент
  появляется в ленте (дозапрос или локальная вставка), textarea чистится, кнопка
  блокируется на время запроса. Пустой/пробельный body не уходит.

## C. Апвоут

- Кнопка `♥ {upvotes}` у карточки. Залогиненный: клик → если ещё не лайкал —
  `insert` в `project_upvotes` `{ project_id, user_id }`; если лайкал — `delete`.
  Начальное состояние «лайкал/нет» — select по `project_id + user_id`.
- **Оптимистично:** счётчик и состояние кнопки меняются сразу, при ошибке запроса —
  откат + короткое сообщение. Счётчик в `projects.upvotes` пересчитает триггер БД —
  локально просто ±1, не перечитывай проект.
- Незалогиненный: клик → модалка логина (`openAuthModal`).
- Защита от дабл-клика: блокируй на время запроса.

## Безопасность (обязательно)
- Весь текст из БД (title, description, body комментов, имена) — только
  `textContent`/`createElement` или `escapeHtml` из `js/util.js`. Комменты — главный
  вектор хранимого XSS, не облажайся.
- `project_url` вставляй через `element.href = value` (не innerHTML). Если протокол
  не http/https — кнопку «Открыть проект» не показывай.
- Никакого `service_role`. ESM + ванильный JS.

## Стиль
- Токены `css/tokens.css`, тёмная тема, моно-акценты, вайб-атрибутика (✦), не шаблонно.
- Мобильная вёрстка обязательна. `prefers-reduced-motion` уважать.

## Проверка (критерии приёмки)
Локально (порт 8080 может быть занят — возьми другой). В БД есть published-проекты (T2).
- Открытие карточки с витрины: контент, инструменты, теги, кнопка «Открыть проект».
- `project.html` без `?id`, с мусорным id, с несуществующим uuid → экран «не найден».
- Комменты: лента читается анонимом; форма — только залогиненным (без логина — гейт
  с модалкой); XSS-тест: коммент `<img src=x onerror=alert(1)>` выводится текстом.
- Апвоут: оптимистичный ±1, повторный клик снимает, состояние переживает перезагрузку
  страницы. Аноним получает модалку.
- Консоль чистая. `npm run lint:js`, `npm run lint:css` — 0 ошибок.
- Что не смог проверить без живой учётки — честно перечисли в отчёте.
