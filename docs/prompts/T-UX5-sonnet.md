## Проблема

На страницах с малым контентом (пустая вкладка `admin.html`, `404.html`, пустые
состояния витрины/профиля) футер не доходит до низа экрана: страница короче
вьюпорта, под футером висит пустой зазор.

## Цель — классический sticky footer (НЕ фиксированный)

- Страница короче экрана → тянется на всю высоту вьюпорта, футер впритык к низу.
- Контента много → футер в конце документа, доступен скроллом. Он НЕ прилипает
  к экрану как шапка.

## Реализация

Все рабочие страницы имеют единую структуру `body > header.header + main +
footer.footer` (проверено: index, projects, project, submit, profile, about,
admin, 404). Поэтому решение — один общий блок в `styles.css`, без правок HTML:

```css
body {
  min-height: 100vh; /* fallback */
  min-height: 100svh;
  display: flex;
  flex-direction: column;
}

main {
  flex: 1 0 auto;
}
```

`100svh` — чтобы на мобилках с динамическими панелями браузера не появлялся
лишний скролл; `100vh` строкой выше — fallback для старых браузеров (незнающие
`svh` просто игнорируют вторую декларацию — порядок строк не менять).

**Обязательно проверь побочные эффекты `display:flex` на `body`:**

1. У `body` есть и другие прямые дети (модалка логина, тосты, скрипты).
   Элементы с `position: fixed/absolute` не участвуют в flex-раскладке — ок.
   Но если найдётся статически позиционированный прямой ребёнок `body` кроме
   header/main/footer — он станет flex-элементом и может растянуться по
   поперечной оси (`align-items: stretch` по умолчанию). Найди всех прямых
   детей `body` на каждой странице и убедись, что раскладка не поехала.
2. Известный грабель проекта: author-`display:flex` перебивает атрибут `hidden`
   (см. CLAUDE.md). К `body` не относится (его не скрывают), но если по ходу
   добавишь новые классы с flex/grid, которые скрываются через `hidden`, —
   сразу гард `.класс[hidden] { display: none; }`.
3. Маргины схлопываться через flex-контейнер перестают (margin-collapsing
   отключается у flex-детей) — проверь, не появились ли лишние/пропавшие
   вертикальные отступы у header/main/footer на страницах.

`archive/oblikflow.html` — легаси-вёрстка без `main` (архив, noindex). Общий
CSS его не заденет (другая структура и свой файл стилей). Открой, убедись,
что ничего не сломалось; чинить прижатие футера там НЕ нужно — страница
архивная, не редизайним.

`me.html` ещё не существует (появится в T14) — пропусти.

## Проверка (критерии приёмки)

Локальный сервер: `python3 -m http.server 8080` (не `file://`).

- Короткие страницы: `404.html`, `admin.html` под гостем («Доступа нет»),
  `profile.html?id=мусор` («не найден») — футер впритык к низу вьюпорта,
  зазора под ним нет.
- Длинные страницы: `index.html`, `about.html`, `project.html` с данными —
  футер в конце документа, скроллом, контент не перекрыт.
- Нет двойного вертикального скролла и горизонтального скролла.
- Десктоп 1280×800 и мобилка 375×812 — обе геометрии.
- Модалка логина открывается/закрывается как раньше (проверь на index).
- Консоль чистая; `npm run lint:js` и `npm run lint:css` — 0 ошибок.
