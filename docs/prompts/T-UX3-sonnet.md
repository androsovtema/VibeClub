## A. Чип категории ≠ кнопка (project.html)

Скрин Тёмы: чип «Продуктивность» и кнопка «Отправить» выглядят одинаково, хотя
категория — это метаданные, а не действие. Принцип: **элементы разного смысла
визуально различаются**.
- Чип категории (сейчас `span.chip`, рендер в `js/project.js` ~стр. 127–133)
  перевести на стиль «метаданных» — согласованный с `.pd-tool-badge`
  (компактный, моно, приглушённый, прямоугольный радиус) вместо
  пилюли-обводки, похожей на кнопку.
- Сделать чип кликабельным: `<a href="projects.html?cat=<ключ>">` — ключ
  категории (`prod`/`biz`/…, НЕ русская подпись; сейчас рендерится
  `CATEGORY_LABELS[tag]`, ключ — сам `tag`). Hover — как у tool-бейджа.
- `js/showcase.js`: на `projects.html` читать `?cat=X` из URL — если ключ
  валиден (есть в `CATEGORY_LABELS`), активировать соответствующий чип
  категории и загрузить с этим фильтром. Комбинация с `?tool=` должна работать
  (оба параметра сразу — оба фильтра).
- Клик по чипу категории на витрине пусть чистит `?cat=` в URL через
  `history.replaceState` (чтобы URL не врал при смене фильтра).

## B. Авторы кликабельны → профиль

Тёма: «могу нажать на ник/аватарку коммента и перейти в профиль?» — сейчас нет.
1. Комменты (`js/project.js`, `renderCommentItem`): аватар и имя автора —
   ссылки на `profile.html?id=<comment.author_id>` (`author_id` уже есть в
   select). Оформление ссылки-имени ненавязчивое: цвет как сейчас,
   подчёркивание/акцент на hover.
2. Шапка карточки проекта: аватар + «Тест Тестов» → `profile.html?id=<author_id>`.
   В `normalizeProject` (`js/projects.js`) добавь `authorId: row.author_id`,
   используй на странице проекта.
3. Не делать ссылками: имена в тексте нот/подписей и автор на карточках витрины
   (там клик по карточке — на проект, вложенные клики не создавать).

## C. Обложка в submit.html: обязательна + стилизованный выбор файла

Скрин Тёмы: нативный «Choose File» без стилей.
1. **Стилизовать выбор файла:** нативный `input[type=file]` не стилизуется —
   стандартный паттерн: input визуально скрыт (но фокусируемый), рядом
   `label`-кнопка в стиле `.btn-secondary`/компакт («Выбрать файл») + текст
   выбранного имени файла. Превью обложки уже есть — не сломай. Фокус-стиль
   на label через `:focus-within`/`:focus-visible` (доступность).
2. **Обложка обязательна:** валидация в `js/submit.js` — без файла форма не
   уходит, ошибка `submit.error.required_cover` («Добавь обложку — без неё
   проект не попадёт в витрину», текст в i18n). Убрать из хинта фразу «Без
   обложки карточка получит фирменный градиент».
3. **Хинт:** «JPEG, PNG или WebP, до 3 МБ. Лучше всего 16:9 — например,
   1280×720» (i18n-ключ `submit.field.cover.hint`). Лимит 3 МБ НЕ менять —
   это осознанное ограничение (бесплатный Supabase Storage 1 ГБ), Тёма в курсе.
4. Градиент-заглушку в рендере карточек НЕ убирать — старые проекты без
   обложки остаются валидными.

## D. Анимация печати терминальных кикеров (index.html)

Тёма: заголовки в стиле промпта («> покажи свежие проекты сообщества»,
«> кто мы») должны печататься, как hero.
- Механика печати уже есть в `main.js` (~стр. 249+, hero-демо: посимвольный
  вывод + sleep). Вынеси переиспользуемый хелпер (например, `typeInto(el,
  text, speed)`) — не дублируй код.
- Применить к `.showcase-kicker` и `.who-kicker`: текст печатается **один раз
  при первом появлении в вьюпорте** (IntersectionObserver, threshold ~0.5,
  после срабатывания — unobserve). До старта текст пустой, но место
  зарезервировано (`min-height`/`min-width` — без прыжка макета).
- `prefers-reduced-motion: reduce` → без анимации, сразу полный текст (как
  сделано в hero).
- Каретка `.typing-caret` уже в разметке — мигает после завершения печати
  (как сейчас), не дублировать.
- Тексты кикеров сейчас в HTML — при переносе в JS положи строки в
  `js/i18n/ru.js`.

## E. Роли в how-terminal заметнее (index.html)

Скрин Тёмы: «> ты», «✦ ИИ», «✓ готово» мелкие и невзрачные.
- `.how-terminal-role`: увеличить (ориентир 15–17px против нынешних ~12),
  вес 600–700, убрать излишнюю приглушённость: «ты» — основной цвет текста,
  «✦ ИИ» — фиолетовый акцент (уже `is-ai`), «✓ готово» — зелёный (уже
  `is-done`), но контраст поднять до читаемого с беглого взгляда.
- Номера 01–04 и тексты шагов не трогать — иерархия: номер → роль → текст.
- Контраст ролей — не ниже WCAG AA на фоне терминала.

## Ограничения

- Токены, i18n для всех новых строк. Пользовательские данные —

- Не трогать: логику комментов (edit/delete), tool-фильтр, сегмент-контрол,
