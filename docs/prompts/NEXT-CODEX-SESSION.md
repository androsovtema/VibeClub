- Не утверждай юридическую квалификацию как гарантию. Для нестабильных или
  юридически значимых фактов используй актуальные официальные источники РКН;

  точный scope и явное разрешение. В фазе РКН изменения БД не нужны.

## Задача A — фактическая матрица для РКН

Подготовь вместе с Тёмой рабочую матрицу по текущему production:

- категории субъектов и персональных данных;
- цели, операции, способы обработки;
- сайт, Auth, БД, Storage, почта, Umami, backup и операторский доступ;
- российский primary storage/processing и фактические меры защиты;
- правила хранения, отзыва согласия и удаления;
- подтверждаемая Тёмой дата начала обработки;
- внешние браузерные получатели до/после T-FRONT-VPS.

Можно создать в `docs/reports/` только обезличенный preflight без реквизитов
оператора и пользователей. Личные поля перечисли Тёме как приватный чек-лист;
значения не записывай.

Для трансграничной передачи проверь реальные network/CSP зависимости. Не
копируй исторические Supabase Cloud, Resend или Umami Cloud как действующих
получателей. До T-FRONT-VPS отдельно учти GitHub Pages/Fastly; после него —
проверь, что они ушли из origin. Как минимум оцени Google Fonts, Cloudflare
Turnstile и внешние аватары. Не превращай технический список в юридическое
заключение без основания.
