## Цель T1
Подключить Supabase к фронту, сделать регистрацию/вход по почте, и модалку логина, которая
открывается по кнопкам «Вступить». Google НЕ используем (заблокирован в РФ). Телефон/Telegram —
не в этой задаче.

## Что сделать

1. **`js/supabase.js`** — инициализация клиента:
   - Импортируй `createClient` из `@supabase/supabase-js` v2 через CDN ESM
     (`https://esm.sh/@supabase/supabase-js@2`).
   - Импортируй ключи из `./config.js`, создай и экспортируй `supabase`.

2. **`js/auth.js`** — функции авторизации (все возвращают `{ data, error }` или бросают
   аккуратно обработанную ошибку):
   - `signUpEmailPassword(email, password, displayName)` — регистрация; `display_name`
     передавай в `options.data` (его подхватит триггер `handle_new_user`).
   - `signInEmailPassword(email, password)`
   - `signInMagicLink(email)` — `signInWithOtp`, `emailRedirectTo = window.location.origin`.
   - `signOut()`, `getCurrentUser()` (из сессии), `onAuthChange(cb)` — подписка.

3. **`js/i18n/ru.js`** — заведи словарь UI-строк (объект `{ key: 'текст' }`) и положи туда
   строки модалки/ошибок. Дальше строки берём отсюда, а не хардкодим. Начни наполнять.

4. **Модалка логина/регистрации** — генерируй её из JS (инжект в `body`), чтобы работала на
   любой странице, подключившей модуль. Требования:
   - Две вкладки: «Войти» и «Регистрация». В регистрации — имя, почта, пароль. Во входе —
     почта, пароль.
   - Кнопка-ссылка «Войти по ссылке на почту» (магик-линк) — поле почты + отправка.
   - Состояния: загрузка (дизейбл кнопки), ошибка (текст по-русски, из `ru.js`), успех.
   - Закрытие: крестик, клик по оверлею, Esc. Доступность: фокус-трап, `aria`-атрибуты.
   - Стиль — на токенах `css/tokens.css`, в духе тёмной темы; не типовой «бутстрап-попап».
     Переиспользуй существующие классы кнопок (`.btn-primary` и т.п.), где уместно.

5. **Подключение и поведение**:
   - Добавь модульный вход `js/app.js` (`<script type="module" src="./js/app.js">`) на
     `index.html` (и на остальных страницах — минимум на index для T1). `app.js` импортирует
     auth и вешает открытие модалки на все `[data-join]`.
   - **Убери из `main.js` старый обработчик `[data-join]`, показывающий тост-заглушку** —
     теперь «Вступить» открывает модалку.
   - Хедер отражает состояние: если залогинен — показать имя и «Выйти» вместо «Вступить»;
     если нет — «Вступить». Обновлять реактивно через `onAuthChange`.
   - После успешного входа/регистрации: закрыть модалку, обновить хедер, показать
     существующий тост об успехе.

## Ограничения
- `service_role`-ключ нигде не использовать. Только `anon` из `config.js`.
- Без тяжёлых библиотек и сборки. Только ESM с CDN + ванильный JS.
- Не ломать текущий визуал и мобильную версию.

## Как проверить (критерии приёмки)
Запусти `python3 -m http.server 8080`, открой `http://localhost:8080/index.html`:
- Клик «Вступить» открывает модалку (не тост).
- Регистрация по почте+паролю создаёт пользователя; в таблице `profiles` появляется строка
  (проверь в Supabase). Магик-линк отправляет письмо со входом.
- После входа хедер показывает имя и «Выйти»; после «Выйти» — снова «Вступить».
- Сессия сохраняется после перезагрузки страницы.
- Ошибки (неверный пароль, занятая почта) показываются по-русски, приложение не падает.
- Прогони линтеры: `npm run lint:js`, `npm run lint:css` — без ошибок.
