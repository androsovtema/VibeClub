## Контекст

В `profiles` уже есть `telegram` и `website`: правятся в `/me`, на `/profile`
рендерятся кликабельными ссылками (см. `js/profile.js`, обработка `@ника` и
`isHttpUrl`). Расширяем до пяти типов контактов: телеграм, сайт, GitHub,
телефон, публичная почта. Цель — связаться с участником в один клик
(коллабы, заказы, команды). Принятое решение: **фиксированный набор колонок,
НЕ произвольный jsonb-список** — проще валидация, чище UI. Не менять это.

## Выход

### 1. Миграция БД

Файл в `supabase/migrations/` с датой в имени, идемпотентный
(`add column if not exists`), + синхронизировать `supabase/schema.sql`.
Применяет Тёма (или Opus через MCP) — отметь в ответе.

```sql
alter table public.profiles
  add column if not exists github       text check (char_length(github) <= 100),
  add column if not exists phone        text check (char_length(phone) <= 30),
  add column if not exists email_public text check (char_length(email_public) <= 200);
```

Плюс добавить недостающие чеки длины на существующие `telegram` (<= 100) и
`website` (<= 300), если их нет (через `alter table … add constraint if not
exists` не бывает — используй `do $$ … $$` с проверкой `pg_constraint` или
`drop constraint if exists` + `add constraint`).

`email_public` — именно отдельная колонка: логин-почту из auth публиковать
без явного ввода нельзя. Ничего не автозаполнять из `auth.users`.

RLS не трогаем: `profiles_update_own` уже пускает владельца к своим колонкам,
триггер `protect_privileged_columns` охраняет только role/is_core.

### 2. `/me` — блок «Контакты»

`me.html` + `js/me.js`: telegram и website уже есть — собрать их и три новых
поля в один блок с заголовком «Контакты» (i18n-строки в `js/i18n/ru.js`):

- телеграм — как сейчас (`@ник` или ник);
- сайт — как сейчас (URL);
- GitHub — ник или полный URL; хранить **ник** (нормализовать: обрезать
  `https://github.com/`, ведущий `@`, слэши);
- телефон — свободный ввод, нормализовать до `+цифры` (убрать пробелы,
  скобки, дефисы); если после чистки не соответствует `^\+?\d{7,15}$` —
  инлайн-ошибка «Проверь номер — цифры и + в начале»;
- почта — простая проверка формата (как в auth-формах), это ПУБЛИЧНАЯ почта,
  плейсхолдер должен намекать, что она видна всем.

Под заголовком блока — тихая строка-предупреждение: «Контакты видны всем
на твоей публичной странице» (i18n).

Сохранение — в тот же update профиля, что и остальные поля. Пустое поле →
`null`, не пустая строка.

### 3. `/profile` — кликабельные чипы контактов

`profile.html` + `js/profile.js`: вместо текущих двух отдельных ссылок —
единый ряд чипов-контактов (стиль — существующие моно-чипы профиля, см.
skills/open_to), рендерятся только заполненные:

- телеграм → `https://t.me/<ник>` (текст `@ник`);
- сайт → сам URL (текст — хост без `https://`);
- GitHub → `https://github.com/<ник>` (текст `<ник>`);
- телефон → `tel:+…` (текст — номер как есть);
- почта → `mailto:…` (текст — адрес).

Внешние ссылки — `target="_blank" rel="noopener noreferrer"` (tel/mailto —
без target). В `select` запроса профиля добавить новые колонки. XSS: значения
вставлять только через `textContent`/безопасные присвоения `href` после
валидации формата (телефон — только `+цифры`, github — только `[\w-]+`,
почта — формат email; иначе чип не рендерить). `javascript:` в website уже
отсекает `isHttpUrl` — не сломать.

Если новый класс чипа получает `display:flex`/`inline-flex` и где-то
скрывается через `hidden` — сразу гард `.класс[hidden] { display: none; }`
(правило проекта).

## Границы

- Не трогать auth, витрину, submit, admin, feedback.
- Произвольные типы контактов, иконки-SVG, копирование в буфер — НЕ делать.
- Тексты — в тоне `docs/03-content.md`, все строки через `js/i18n/ru.js`.

## Приёмка

- Заполнил все 5 полей в `/me` → сохранилось, после перезагрузки на месте.
- `/profile`: 5 чипов, каждый открывает то, что должен (телеграм, сайт,
  гитхаб — новая вкладка; tel:/mailto: — системный обработчик).
- Пустые поля → чипов нет, ряд не рендерится совсем, если пусто всё.
- `https://github.com/ник/` и `@ник` в поле GitHub дают один результат.
- `+7 (999) 123-45-67` сохраняется как `+79991234567`, чип звонит.
- Кривой телефон/почта — инлайн-ошибка, сохранение блокируется.
- В БД лишние символы не попадают (проверить строку в таблице).
- 375px ок; консоль чистая; линтеры 0; `npm run security-check` зелёный.
- Напоминание в ответе: применить миграцию.
