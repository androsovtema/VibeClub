## Цель
Убрать последний фейк на главной — захардкоженные `@ники` + «+1 236 ещё»

реальных участников. Плюс публичная страница профиля.

## A. Публичный профиль — `profile.html?id=...` + `js/profile.js`

- Шапка/футер — как на `project.html` (nav + `data-header-auth`, `js/app.js`).
- Данные: `profiles` по id (select открыт анониму по RLS) — аватар (или инициалы,
  переиспользуй `initialOf` из `js/projects.js`), `display_name`, `bio`,
  ссылки `telegram` / `website` (если заполнены).
- **Безопасность ссылок:** `website` вставлять только если протокол http/https
  (паттерн `isHttpUrl` уже есть в `js/project.js` — вынеси в `js/util.js`, не копипасть).
  `telegram` — это хэндл: рендерь как `https://t.me/{handle}` (символ `@` отрезать),
  сам хэндл — `textContent`.
- **Проекты участника:** сетка published-проектов этого автора. Расширь
  `fetchPublishedProjects` опцией `authorId` (`.eq('author_id', id)`) — сигнатуру для
  витрины не ломай. Карточки — `renderProjectCard`, пусто — «Пока без проектов».
- Плохой/несуществующий id → экран «не найден» (как на `project.html`) со ссылкой
  на витрину.
- Все строки — `js/i18n/ru.js`. Тайтл: `{display_name} — We Designerz`.

## B. Лента «уже с нами» на главной

- Удалить фейковый блок из `main.js` (см. выше) целиком.
- Новый модуль `js/members.js`: тянет из `profiles` до ~24 участников
  (`display_name`, `avatar_url`, `id`, сортировка `created_at desc`), рендерит в
  `#member-chips` ряд чипов: аватар-кружок (или инициалы) + имя. Каждый чип —
  ссылка на `profile.html?id={id}`.
- **Самолистание:** если участников достаточно (ряд шире контейнера) — плавная
  бесконечная CSS-прокрутка (marquee: дублируй ряд, `transform` в `@keyframes`);
  участников мало — статичный ряд, без пустой прокрутки. При
  `prefers-reduced-motion` — всегда статично.
- **Честность:** никаких «+N ещё» с выдуманным N. Если участников < 3 — покажи
  только реальных, без приписок. Ошибка сети → блок скрыт (паттерн `js/stats.js`).
- Рендер — `createElement`/`textContent` (имена — пользовательские данные, XSS).

## C. Живой тост «недавно в витрине» (по 06, «живые события»)

- На главной, раз за сессию (`sessionStorage`), через пару секунд после загрузки:
  тост «{автор} добавил «{title}»» по самому свежему published-проекту (данные уже
  есть в выдаче витрины — не делай лишний запрос, возьми из результата `showcase.js`
  или отдельным лёгким запросом `limit 1`). Клик по тосту → страница проекта.
- Реальные данные, никакой выдумки. БД пустая → тоста нет.
- Разметка `#join-toast` уже есть (`main.js`/`app.js` показывают через неё ошибки/
  успехи) — не ломай её; лучше отдельный тост-элемент с тем же стилем.
- `aria-live="polite"`, автоскрытие ~6 сек, `textContent` для данных.
