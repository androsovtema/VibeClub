# 05 — Запуск и сидинг

## Правило №1: не запускать пустым

Пустая витрина и пустой чат убивают сообщество в первый день. Зашедший с YouTube должен
увидеть **живое** место: проекты уже есть, в чате уже разговаривают. Сидинг — обязательный
этап перед публичным анонсом, не опциональный.

## Сидинг витрины (до анонса)

Цель: ≥ 10–15 проектов в витрине на момент публикации видео.
- **Флагманы We Designerz** (`is_core = true`): проекты, что Тёма уже навайбкодил —
  включая **сам этот сайт** (мета-пример: «сообщество вайбкодеров, само навайбкоженное»).
  Всё, что показывалось на канале.
- **Проекты Алины и команды**, которые подходят под формат.
- **Первые участники:** до анонса позвать 10–20 человек из активной аудитории канала /
  лички, попросить добавить их проекты. Лично, не публично.

## Сидинг чата (до анонса)

- Создать Telegram-чат, оформить (название, описание, закреп с правилами и ссылкой на сайт).
- Завести ядро: Тёма, Алина, Андрей + первые 10–20 приглашённых.
- Заготовить 3–5 стартовых тем для обсуждения, чтобы чат не выглядел молчащим.
- Правила: коротко, дружелюбно — делись проектами, помогай, без спама/токсичности.

## Воронка (как это работает на запуске)

```
Видео на YouTube  →  сайт (витрина живая)  →  [Вступить] → Telegram-чат
                                            →  [Добавить проект] → submit
```
Канал = двигатель трафика. Каждое следующее видео может звать на сайт и в чат.

## Чек-лист готовности к анонсу

- [ ] T0–T8 из `04-tasks-sonnet.md` выполнены и приняты.
- [ ] RLS проверен вручную: аноним не пишет, чужое не редактируется (см. критерии T0).
- [ ] Логин email (пароль + магик-линк) работает на проде (`wedesignerz.com`).
      (Google не делаем — заблокирован в РФ, см. T1.)
- [ ] **Свой SMTP подключён** (Resend/Brevo, бесплатного тарифа хватит): встроенная
      почта Supabase шлёт ~2 письма/час на весь проект — реальные регистрации упрутся
      в лимит (Тёма поймал это уже на тестах, 2026-07-02). Dashboard → Project
      Settings → Auth → SMTP Settings. Тест: регистрация + магик-линк доходят.
- [ ] Витрина засеяна (≥ 10–15 проектов, среди них флагманы We Designerz).
- [ ] Telegram-чат создан, оформлен, ядро внутри, есть стартовые темы.
- [ ] Кнопка «Добавить проект» проходит полный путь: логин → форма → модерация → публикация.
- [ ] Сообщения под проектом работают (чтение всеми, запись залогиненными).
- [ ] Мобильная версия и адаптив проверены.
- [ ] SEO/OG обновлены, превью при шеринге корректное.
- [ ] Старый студийный оффер убран, hire-CTA ровно один (в «Кто мы»).
- [ ] Прогон «глазами новичка с YouTube»: за 5 секунд понятно, что это и что делать.

## Заготовка скрипта видео-анонса

Структура ролика (черновик, Тёма доводит под свой стиль):
1. **Хук:** «Я навайбкодил сайт для сообщества вайбкодеров — и сейчас покажу, зачем он тебе.»
2. **Проблема:** раньше, чтобы создать продукт, надо было уметь кодить. Теперь — нет.
3. **Идея сообщества:** We Designerz — место для идейных, кто хочет создавать с ИИ.
4. **Демо сайта:** витрина проектов → открыть проект → сообщения → добавить свой → вступить.
5. **Призыв:** ссылка на сайт и чат в описании. «Покажи, что создаёшь ты.»
6. **Честно про нас:** мы — дизайнеры-практики, вот портфолио; но главное здесь — вы и ваши
   проекты.

## После запуска (первые 30 дней)

- Следить за метриками из `00-vision.md` (участники, проекты, сообщения).
- Каждое новое видео = повод звать на сайт/в чат и подсвечивать лучшие проекты сообщества.
- Раз в неделю — «проект недели» из витрины (контент для канала + мотивация участникам).
- Когда наберутся реальные цифры — только тогда думать о выходе на институты/инвесторов
  (не раньше, см. `00-vision.md`).
