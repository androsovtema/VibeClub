# 02 — Информационная архитектура

## Карта страниц

```
/ (index.html)            Главная. Ядро = сообщество.
/projects.html            Полная витрина проектов сообщества + фильтры + «Добавить»
/project.html?id=...      Карточка проекта + сообщения (или модалка из витрины)
/submit.html              Форма добавления проекта (требует логин)
/me.html                  Профиль участника (этап 2, не критично для запуска)
/admin.html               Модерация (видна только роли admin)
/about.html               «Кто мы» + дизайн-портфолио (доказательство) + hire CTA
/projects/*.html          Существующие дизайн-кейсы — оставляем как есть, линкуем из about
```

Навигация в хедере: **Сообщество (главная) · Проекты · Кто мы · [Вступить]**
`[Вступить]` — акцентная кнопка → Telegram.

## Структура главной (`/`) — сверху вниз

Один экран = одна мысль. Порядок выстроен под единственное действие «вступить и показать».

1. **Hero**
   - H1: суть в одну строку — We Designerz как сообщество создающих с ИИ.
   - Подзаголовок: для кого и зачем.
   - Два CTA: `[Вступить]` (→ Telegram, главный) и `[Добавить проект]` (→ submit).
   - Лёгкая динамика как сейчас (анимация слова) — стиль сохраняем.

2. **Витрина проектов сообщества (главный контент)**
   - Лента карточек из Supabase (`published`), сначала свежие/топ.
   - Карточка: обложка, название, автор, теги, апвоуты, кол-во сообщений.
   - Кнопка «Все проекты» → `/projects.html`.
   - **Никогда не пустая** — на запуске засеяна (см. 05-launch.md).

3. **Как это работает (3 шага)**
   - Придумай → Навайбкодь с ИИ → Покажи сообществу.
   - Переосмысление текущего блока «Как мы работаем».

4. **Манифест (короткий)**
   - Про идейных людей, которые раньше не могли, а теперь могут. Переиспользует энергию
     текущего блока «Вы можете сделать это сами» — но развёрнут к созиданию, не к найму.

5. **Кто мы + доказательство экспертизы**
   - Тёма, Алина, Андрей — лица и роли (честно, по реальному весу).
   - Полоска дизайн-портфолио (карусель/логотипы клиентов уже есть) как доказательство.
   - **Единственный** hire-CTA: «Нужен дизайн под проект? Напишите» → Telegram личка.
   - Ссылка «Смотреть кейсы» → `/about.html` / `projects/*.html`.

6. **Финальный призыв вступить**
   - Повтор главного CTA `[Вступить]` + ссылки на YouTube.

7. **Footer**
   - We Designerz · YouTube · Telegram · (позже — соцсети участников).

## Структура `/projects.html`

- Заголовок + краткое описание витрины.
- Фильтры: по тегам, по инструментам, «только флагманы We Designerz» (`is_core`), сортировка
  (свежие / по апвоутам).
- Сетка карточек (пагинация или бесконечная подгрузка).
- Заметная кнопка `[Добавить свой проект]` → `/submit.html`.

## Структура `/project.html?id=...`

- Обложка, название, автор (ссылка на профиль), теги, инструменты, ссылка «Открыть проект».
- Апвоут.
- Описание (как сделано).
- **Сообщения** — лента комментариев + форма (для залогиненных).

## Структура `/submit.html`

- Гейт: не залогинен → кнопки входа (Google / email magic link).
- Форма: название, описание, ссылка на проект, обложка (загрузка), теги, инструменты.
- После отправки: «Проект отправлен на модерацию, появится после проверки».

## Структура `/about.html` («Кто мы»)

- Развёрнутая история We Designerz и людей.
- Дизайн-портфолио: существующие кейсы как доказательство.
- Hire-CTA (тот самый единственный) — здесь он уместен в полном виде.

## Принципы

- **5 секунд правило:** зашедший с YouTube за 5 сек понимает — «это сообщество, где
  показывают и обсуждают навайбкоженные проекты, и я могу вступить».
- **Один главный CTA** на всех страницах — «Вступить». Hire-CTA — ровно один, в зоне «Кто мы».
- **Контент важнее текста о себе:** живые проекты сообщества выше рассказа про команду.
- Визуальный стиль сохраняем: шрифты Cormorant Garamond + Inter, текущая эстетика, анимации.
