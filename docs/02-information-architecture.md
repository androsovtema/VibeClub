# 02 — Информационная архитектура

## Карта страниц

```
/ (index.html)            Главная. Ядро = сообщество.
/projects.html            Полная витрина проектов сообщества + фильтры + «Добавить»
/project.html?id=...      Карточка проекта + сообщения (или модалка из витрины)
/submit.html              Форма добавления проекта (требует логин)
/me.html                  Профиль участника (этап 2, не критично для запуска)
/admin.html               Модерация (видна только роли admin)
/about.html               «Кто мы» + дизайн-портфолио (доказательство) + hire CTA
/projects/*.html          Существующие дизайн-кейсы — оставляем как есть, линкуем из about
```

Навигация в хедере: **Главная · Проекты · О нас · [Вступить]**
`[Вступить]` — акцентная кнопка → **auth-модалка** (вступление в клуб = профиль
на платформе, см. `00-vision.md` «Семантика "Вступить"» и `10-membership.md`).
Для залогиненного участника кнопка превращается в «Добавить проект».
Telegram-чат — отдельная второстепенная ссылка (общение), не главный CTA.

## Структура главной (`/`) — сверху вниз

Один экран = одна мысль. Порядок выстроен под единственное действие «вступить и показать».

1. **Hero**
   - H1: суть в одну строку — We Designerz как сообщество создающих с ИИ.
   - Подзаголовок: для кого и зачем.
   - Два CTA: `[Вступить]` (→ auth-модалка, главный) и Telegram-чат (второстепенный).
   - Счётчики участников/проектов — только после порога (`00-vision.md`,
     «Социальное доказательство»). До порога — метрики канала / без счётчиков.
   - Лёгкая динамика как сейчас (анимация слова) — стиль сохраняем.

2. **Витрина проектов сообщества (главный контент)**
   - Лента карточек из Supabase (`published`), сначала свежие/топ.
   - Карточка: обложка, название, автор, теги, апвоуты, кол-во сообщений.
   - Кнопка «Все проекты» → `/projects.html`.
   - **Без фильтров-чипов на главной** (правка по критике 2026-07-05): при малом
     числе проектов клик по категории даёт пустое состояние «здесь пока пусто»
     прямо на главной — антисоциальное доказательство. Фильтры живут на
     `/projects.html`; на главной — только лента + «Все проекты →».
   - **Никогда не пустая** — на запуске засеяна (см. 05-launch.md).

3. **Как это работает (3 шага)**
   - Придумай → Навайбкодь с ИИ → Покажи сообществу.
   - Переосмысление текущего блока «Как мы работаем».

4. **Манифест (короткий)**
   - Про идейных людей, которые раньше не могли, а теперь могут. Переиспользует энергию
     текущего блока «Вы можете сделать это сами» — но развёрнут к созиданию, не к найму.

5. **Кто мы + доказательство экспертизы**
   - Тёма, Алина, Андрей — лица и роли (честно, по реальному весу).
   - Полоска дизайн-портфолио (карусель/логотипы клиентов уже есть) как доказательство.
   - **Единственный** hire-CTA: «Нужен дизайн под проект? Напишите» → Telegram личка.
   - Ссылка «Смотреть кейсы» → `/about.html` / `projects/*.html`.

6. **Финальный призыв вступить**
   - Повтор главного CTA `[Вступить]` + ссылки на YouTube.

7. **Footer**
   - We Designerz · YouTube · Telegram · (позже — соцсети участников).

## Структура `/projects.html`

- Заголовок + краткое описание витрины.
- Фильтры: по тегам, по инструментам, «только флагманы We Designerz» (`is_core`), сортировка
  (свежие / по апвоутам).
- Сетка карточек (пагинация или бесконечная подгрузка).
- Заметная кнопка `[Добавить свой проект]` → `/submit.html`.

## Структура `/project.html?id=...`

- Обложка, название, автор (ссылка на профиль), теги, инструменты, ссылка «Открыть проект».
- Апвоут.
- Описание (как сделано). После T12 — структура «что это / чем вайбкодил /
  что было сложно» вместо одного свободного поля: ценность страницы для ЦА —
  в «как сделано», свободное поле её не вытягивает.
- **Сообщения** — лента комментариев + форма (для залогиненных).
- Ограничение статики: OG-теги у всех проектов одинаковые — авторский шаринг
  проекта даёт generic-превью. Минимум — сильная общая OG-картинка (T8);
  pre-render/edge-решение — P2 (`09-growth-plan.md`).

## Структура `/submit.html`

- Гейт: не залогинен → объяснение «что/как/зачем» (T10) + кнопка вступления.
  Кнопка гейта — **«Вступить или войти»**, не «Войти»: новичок ещё не имеет
  аккаунта, слово «войди» читается как барьер «только для своих».
  (Google-вход не делаем — заблокирован в РФ, см. `05-launch.md`; вход =
  почта/пароль + magic link.)
- Форма: название, описание, ссылка на проект, обложка (загрузка), теги, инструменты.
- После отправки: «Проект отправлен на модерацию, появится после проверки».

## Структура `/about.html` («Кто мы»)

- Развёрнутая история We Designerz и людей.
- Дизайн-портфолио: существующие кейсы как доказательство.
- Hire-CTA (тот самый единственный) — здесь он уместен в полном виде.

## Принципы

- **5 секунд правило:** зашедший с YouTube за 5 сек понимает — «это клуб, где
  показывают и обсуждают навайбкоженные проекты, и я могу вступить».
- **Один главный CTA** на всех страницах — «Вступить» (= членство, см. `00-vision.md`).
  Hire-CTA — один смысловой оффер с двумя входами: тизер в «Кто мы» на главной и
  полный вид на `/about.html`. Больше нигде не размножать.
- **Контент важнее текста о себе:** живые проекты сообщества выше рассказа про команду.
- Визуальный стиль — направление A «Электрический холст» (`06-design-direction.md`):
  Onest + JetBrains Mono, тёмная база, градиентные акценты.

## Словарь терминов (держать единый, правка по критике 2026-07-05)

Сейчас терминология скачет: «Проекты» (нав) / «витрина сообщества» (кикер) /
«Что навайбкодили» (H2) / «Витрина проектов» (футер). Фиксируем:

| Понятие | Слово везде |
|---|---|
| Совокупность опубликованных проектов | **витрина** |
| Пункт навигации и заголовок страницы | **Проекты** |
| Член сообщества | **участник** (в копирайте идентичности — «вайбкодер») |
| Сообщество как целое | **клуб** (в текстах), «сообщество» (нейтральный синоним) |
| Работы команды в витрине | **флагманы** (`is_core`, метка ✦) |

H2-заголовки секций могут быть образными («Что навайбкодили») — это голос бренда,
не термин. Но служебные подписи (кикеры, футер, кнопки) используют слова из таблицы.

## Гигиена корня сайта

В корне живут только страницы платформы. Старые клиентские лендинги
(`oblikflow.html` — «яхты в Сочи») из корня убрать: в архив/отдельную папку вне
индексации либо удалить. Они индексируются, размывают позиционирование и SEO.
